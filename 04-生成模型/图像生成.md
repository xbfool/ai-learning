# 图像生成

> 从 Stable Diffusion 到 FLUX.2，图像生成经历了架构范式的根本转变。

## 一、架构演进时间线

```
2022: Stable Diffusion 1.x → U-Net + DDPM + CLIP
2023: SDXL → 更大 U-Net + 双文本编码器
2024: SD3 → MMDiT (Transformer 替代 U-Net) + Flow Matching
2024: FLUX.1 → 双流 DiT + Rectified Flow
2025: FLUX.2 → 32B 参数 DiT + VLM 耦合
2025: GPT-4o → 纯自回归图像生成 ← 范式转变
```

## 二、核心架构

### DiT (Diffusion Transformer)
- **论文**：Peebles & Xie (2023)
- 用 Vision Transformer 替代 U-Net 做扩散去噪
- **优势**：遵循 Scaling Laws，模型越大效果越好
- **劣势**：计算量大于 U-Net
- 被 FLUX, SD3, Sora 等所有 SOTA 模型采用

### Flow Matching / Rectified Flow
- **核心**：学习从噪声到数据的直线路径 (OT path)
- **替代**：传统 DDPM 的弯曲扩散路径
- **优势**：
  - 轨迹更直 → 需要更少的采样步骤
  - 更容易蒸馏到 1-4 步生成
  - 训练目标更简单 (velocity matching)
- 被 FLUX, SD3, 大部分视频模型使用

### 自回归图像生成 (AR)
- **核心**：将图像 token 化，用 LLM 方式逐 token 生成
- **代表**：GPT-4o Images, Emu3, LlamaGen, VAR
- **优势**：
  - 与文本生成统一架构
  - 指令遵循能力强
  - 文字渲染准确
  - 多轮编辑自然
- **劣势**：高频细节不如扩散模型

### 混合架构
- **Transfusion** (Meta)：文本自回归 + 图像扩散，共享一个 Transformer
- **GLM-Image** (智谱)：自回归骨干 + 扩散解码器，中文/多语言排版出色
- **Show-o**：自回归文本 + 离散扩散图像

## 四、训练方法论

### 数据处理
1. **合成标注** (Synthetic Captioning)：用 VLM 对训练图像重新生成详细描述
   - DALL-E 3 首创此方法
   - 显著提升 prompt 遵循度
2. **渐进式训练**：低分辨率 → 高分辨率，逐步提升
3. **多分辨率/多比例训练**：适应不同输出尺寸

### 对齐 (Alignment)
- **Diffusion-DPO**：将 DPO 应用到扩散模型
- **奖励模型微调**：用人类偏好奖励模型指导扩散模型
- **CFG (Classifier-Free Guidance)**：仍是标准技术，但部分模型（FLUX schnell）已不需要

### 加速
- **蒸馏**：将多步扩散蒸馏为 1-4 步
- **一致性模型 (Consistency Models)**：直接预测干净图像
- **对抗蒸馏**：用 GAN loss 辅助蒸馏

## 六、核心论文深度解读

> 以下章节对图像生成领域的关键论文进行深入剖析，涵盖架构设计、数学原理和工程实现。

### 6.1 DiT：Diffusion Transformer

> 论文：*Scalable Diffusion Models with Transformers* (Peebles & Xie, ICCV 2023)

#### 6.1.1 核心动机：为什么要用 Transformer 替代 U-Net？

传统扩散模型（如 Stable Diffusion 1.x/2.x、DALL-E 2）的骨干网络是 U-Net，一种具有跳跃连接的编码器-解码器卷积架构。U-Net 在图像生成中表现优秀，但存在根本性局限：

1. **缺乏清晰的 Scaling Laws**：增大 U-Net 的通道数或深度，效果提升不稳定、不可预测。
2. **归纳偏置过强**：卷积的局部感受野限制了全局上下文建模。
3. **架构异构性**：下采样、上采样、跳跃连接使得模型结构复杂，难以系统化地扩展。

DiT 的核心贡献是证明：**将扩散模型的骨干完全替换为标准 Vision Transformer (ViT)，可以获得清晰的 Scaling Laws**——模型越大、token 越多，FID 越低，与 LLM 领域的规律一致。这一发现直接催生了后续的 SD3、FLUX、Sora 等所有 SOTA 模型。

#### 6.1.2 Patchification（图像块化）

DiT 不直接处理原始像素，而是在预训练 VAE 的潜空间中工作：

```
原始图像 256x256x3
    ↓ VAE Encoder
潜空间表示 32x32x4   (空间缩小 8 倍，通道 4)
    ↓ Patchify
Token 序列           (展平为 Transformer 输入)
```

**Patchify 层**是 DiT 的第一层，将潜空间的空间维度切分为 patch，每个 patch 通过线性投影变为一个 d 维的 token：

| Patch 大小 p | Token 数量 (32x32 潜空间) | 计算量关系 |
|:---:|:---:|:---:|
| p=2 | (32/2)^2 = 256 个 token | 基准 |
| p=4 | (32/4)^2 = 64 个 token | ~1/4 计算量 |
| p=8 | (32/8)^2 = 16 个 token | ~1/16 计算量 |

**关键发现**：patch 越小（token 越多），模型性能越好，但计算量也越大。将 patch 大小减半，token 数量翻 4 倍，总 GFLOPs 至少翻 4 倍。Patchify 之后，对所有 token 施加标准的正弦余弦位置编码 (sinusoidal positional embeddings)。

#### 6.1.3 AdaLN-Zero 条件注入机制

DiT 探索了四种将条件信息（时间步 t、类别标签 c）注入 Transformer 的方式：

| 方案 | 方法 | FID (400K 步) | 额外 GFLOPs |
|:---|:---|:---:|:---:|
| In-context | 将 t、c 作为额外 token 拼接到序列中 | 最差 | 可忽略 |
| Cross-attention | 在每个 block 中加入交叉注意力层 | 中等 | 较多 |
| adaLN | 自适应层归一化，用 (t+c) 回归 gamma/beta | 较好 | 最少 |
| **adaLN-Zero** | adaLN + 零初始化残差缩放 | **最好** | 最少 |

**adaLN-Zero 的具体做法**：

1. 将时间步 t 的嵌入与类别标签 c 的嵌入相加，得到条件向量。
2. 用条件向量通过 MLP 回归出自适应层归一化的 6 个参数：gamma_1、beta_1（注意力层前的 LN）、gamma_2、beta_2（前馈层前的 LN）以及两个残差缩放系数 alpha_1、alpha_2。
3. **零初始化**：将每个 DiT block 的残差缩放系数 alpha 初始化为 0，使得模型在训练开始时每个 block 等价于恒等函数。

```
输入 x
  ↓
LayerNorm → 用 (gamma_1, beta_1) 调制
  ↓
Self-Attention
  ↓
× alpha_1 (初始为 0)  ← 零初始化
  ↓
+ x (残差连接)
  ↓
LayerNorm → 用 (gamma_2, beta_2) 调制
  ↓
Feed-Forward
  ↓
× alpha_2 (初始为 0)  ← 零初始化
  ↓
+ (残差连接)
  ↓
输出
```

adaLN-Zero 在 400K 训练步时 FID 接近 in-context 方案的一半，同时是计算效率最高的方案。这一设计后来成为几乎所有扩散 Transformer 的标准选择。

#### 6.1.4 Scaling Laws 与模型配置

DiT 定义了四种标准模型尺寸，与 ViT 对齐：

| 模型 | 层数 N | 隐藏维度 d | 注意力头数 | 参数量 | GFLOPs (p=2) |
|:---|:---:|:---:|:---:|:---:|:---:|
| DiT-S | 12 | 384 | 6 | 33M | 6.1 |
| DiT-B | 12 | 768 | 12 | 130M | 23.0 |
| DiT-L | 24 | 1024 | 16 | 458M | 80.7 |
| DiT-XL | 28 | 1152 | 16 | 675M | 118.6 |

**核心发现**（Scaling Laws）：

- 在 patch 大小固定时，增大模型（S → B → L → XL），FID 单调下降。
- 在模型固定时，减小 patch 大小（增加 token 数量），FID 单调下降。
- **GFLOPs 与 FID 存在强相关性**：总计算量是最有效的性能预测指标，无论是通过增大模型还是增加 token 数量来提升 GFLOPs。
- DiT-XL/2（最大模型 + 最小 patch）达到 ImageNet 256x256 上 FID=2.27，与当时最好的扩散模型 (LDM-4) 持平或更优。

这一发现的意义在于：**扩散模型现在可以像 LLM 一样通过简单地"堆算力"来系统化提升效果**，为后续 SD3 (2B)、FLUX (12B)、Sora 等大模型奠定了理论基础。

### 6.2 Flow Matching 与 Rectified Flow

> 核心论文：
> - *Flow Matching for Generative Modeling* (Lipman et al., ICLR 2023)
> - *Flow Straight and Fast: Learning to Generate and Transfer Data with Rectified Flow* (Liu et al., ICLR 2023)
> - *Improving the Training of Rectified Flows* (Esser et al., 2024 - SD3 论文)

#### 6.2.1 从扩散到流匹配：概念转变

**扩散模型 (DDPM)** 的路径是弯曲的：
- 前向过程：逐步加噪 x_0 → x_1 → ... → x_T（高斯过程，路径弯曲）
- 反向过程：学习预测噪声 epsilon，逐步去噪
- 缺点：需要 20-50 步才能获得高质量结果，路径弯曲导致离散化误差大

**流匹配 (Flow Matching)** 的核心思想完全不同：
- 不加噪/去噪，而是学习一个**速度场 (velocity field)** v(x, t)
- 该速度场定义了一条从噪声分布 p_0 到数据分布 p_1 的连续变换路径
- 沿速度场积分 ODE 即可完成生成

#### 6.2.2 数学基础

**概率路径 (Probability Path)**：

设 p_t(x) 是从源分布 p_0（噪声）到目标分布 p_1（数据）的插值概率密度，满足连续性方程：

```
∂p_t/∂t + ∇·(p_t · u_t) = 0
```

其中 u_t(x) 是生成该概率路径的速度场。

**流匹配目标 (Flow Matching Objective)**：

```
L_FM(theta) = E_{t~U(0,1), x~p_t(x)} || v_theta(x, t) - u_t(x) ||^2
```

即训练神经网络 v_theta 去匹配/回归真实速度场 u_t。但问题在于 u_t(x) 通常不可直接计算。

**条件流匹配 (Conditional Flow Matching, CFM)**：

关键突破是：不直接匹配全局速度场 u_t，而是匹配**条件速度场** u_t(x | x_1)。对于一条从 x_0 到 x_1 的条件路径：

```
条件插值路径:  x_t = (1-t) · x_0 + t · x_1    (线性插值)
条件速度场:    u_t(x | x_0, x_1) = x_1 - x_0   (常数，与 t 无关！)
```

**CFM 训练目标**：

```
L_CFM(theta) = E_{t~U(0,1), x_0~p_0, x_1~p_1} || v_theta(x_t, t) - (x_1 - x_0) ||^2
```

这个目标极其简洁：采样一对 (x_0, x_1)，在 t 时刻取线性插值点 x_t，训练网络预测方向 (x_1 - x_0)。与 DDPM 的 epsilon 预测相比，训练目标更加直观——直接预测从噪声到数据的"速度"。

#### 6.2.3 最优传输 (Optimal Transport) 路径

条件流匹配中如何选择 (x_0, x_1) 的配对方式至关重要：

- **独立耦合**：x_0 和 x_1 独立采样（最简单，但路径可能交叉）
- **最优传输 (OT) 耦合**：找到最小化传输代价的配对

```
OT 耦合: π* = arg min_π ∫ ||x_1 - x_0||^2 dπ(x_0, x_1)
```

OT 耦合使得路径更直、交叉更少，这在 minibatch 级别可以近似实现（minibatch OT）。更直的路径意味着：
- 数值积分时离散化误差更小
- 可以用更少的步数（甚至 1 步）完成生成

#### 6.2.4 Rectified Flow：路径矫正

Rectified Flow 在流匹配的基础上提出了 **Reflow** 算法来进一步拉直路径：

**第一步——Rectified Flow (1-Rectified Flow)**：
```
X_t = (1-t) · X_0 + t · X_1,  X_0 ~ N(0,I),  X_1 ~ p_data
训练 v_theta 使得 L = E || v_theta(X_t, t) - (X_1 - X_0) ||^2
```

**第二步——Reflow（路径矫正）**：
1. 用已训练好的 v_theta，从随机噪声 Z_0 ~ N(0,I) 出发，沿 ODE 积分得到 Z_1。
2. 现在 (Z_0, Z_1) 形成新的配对——与独立采样不同，这对配对已经通过学到的流相连。
3. 用这些新配对重新训练一个 flow：2-Rectified Flow。
4. 理论保证：每次 Reflow，路径的"弯曲程度"（transport cost）单调递减。

```
1-Rectified Flow: 路径有交叉、弯曲
    ↓ Reflow
2-Rectified Flow: 路径更直
    ↓ Reflow
3-Rectified Flow: 路径近乎直线
    ↓ 蒸馏
1 步生成！
```

**为什么路径更直 = 步数更少？**

直线路径的 ODE 可以精确地用单步欧拉法模拟（因为速度场恒定）。弯曲路径则需要多步才能准确追踪。在 CIFAR-10 上，2-Rectified Flow 仅用 1 步欧拉法即可达到 FID=4.85，优于当时所有一步蒸馏方法。

#### 6.2.5 Flow Matching vs DDPM 对比总结

| 特性 | DDPM | Flow Matching / Rectified Flow |
|:---|:---|:---|
| 路径形状 | 弯曲（高斯扩散过程） | 近直线（线性插值 + Reflow） |
| 训练目标 | 预测噪声 epsilon | 预测速度 v = x_1 - x_0 |
| 采样步数 | 20-50 步 | 1-10 步（路径越直步数越少） |
| 蒸馏难度 | 较难（弯曲路径难以一步近似） | 容易（直线路径天然适合一步生成） |
| 数学框架 | 随机微分方程 (SDE) | 常微分方程 (ODE) |
| 采用者 | SD 1.x, SDXL | SD3, FLUX, Sora, 大部分新模型 |

### 6.3 FLUX 架构详解

> FLUX.1 (Black Forest Labs, 2024.08)，12B 参数

FLUX 是由 Stable Diffusion 核心团队离开 Stability AI 后创立的 Black Forest Labs 开发的模型。它将 DiT 和 Flow Matching 的思想推进到了工程极致。

#### 6.3.1 整体架构

```
输入文本 ──→ [CLIP ViT-L] ──→ 文本 token (pooled embedding 用于 timestep 调制)
         ──→ [T5-XXL]     ──→ 文本 token 序列 (详细语义)

输入噪声 ──→ [VAE Encoder] ──→ 潜空间 ──→ [Patchify] ──→ 图像 token 序列

          ┌─────────────────────────────────────┐
          │   19 个 Double-Stream Blocks         │  ← 文本和图像分别处理
          │   (文本流 & 图像流独立权重)            │     但在注意力层联合
          ├─────────────────────────────────────┤
          │   38 个 Single-Stream Blocks         │  ← 文本和图像合并为
          │   (统一序列处理)                       │     单一序列处理
          └─────────────────────────────────────┘
                      ↓
               [线性投影] → 输出潜空间
                      ↓
               [VAE Decoder] → 生成图像
```

**参数分布**：12B 参数中，19 个 Double-Stream Blocks + 38 个 Single-Stream Blocks 构成了模型的主体。

#### 6.3.2 Double-Stream Blocks（双流 Transformer 块）

这是 FLUX 最核心的创新。在每个 Double-Stream Block 中：

```
文本 token 序列                    图像 token 序列
     ↓                                ↓
  [LayerNorm + 调制]              [LayerNorm + 调制]
     ↓                                ↓
  Q_text, K_text, V_text          Q_img, K_img, V_img
     ↓                                ↓
     └──── 拼接 ────→ [联合注意力 (Joint Attention)] ←─── 拼接 ────┘
                              ↓
                    拆分回文本和图像
     ↓                                ↓
  [文本 MLP]                      [图像 MLP]
     ↓                                ↓
  更新的文本 token                 更新的图像 token
```

关键设计：
- **独立权重**：文本和图像使用各自独立的 QKV 投影矩阵和 MLP，因为两种模态的表征空间差异很大。
- **联合注意力**：将文本和图像的 Q、K、V 拼接后做一次统一的注意力计算，使得两种模态可以互相"看到"对方。
- 这与 SD3 的 MMDiT 设计一脉相承，但 FLUX 在此基础上增加了 Single-Stream 阶段。

#### 6.3.3 Single-Stream Blocks（单流 Transformer 块）

经过 19 层双流处理后，文本和图像 token 被**拼接为一个统一序列**，送入 38 层标准 Transformer block：

```
[文本 token_1, ..., 文本 token_n, 图像 token_1, ..., 图像 token_m]
                            ↓
                  标准 Transformer Block
                  (Self-Attention + MLP)
                            ↓
                  统一处理后的序列
```

这一阶段的文本和图像 token 共享同一组权重，允许更深层次的跨模态融合。

#### 6.3.4 位置编码与其他技术细节

- **旋转位置编码 (RoPE)**：FLUX 使用 Rotary Positional Embeddings，通过在复数平面上旋转 Q 和 K 向量来编码相对位置。相比绝对位置编码，RoPE 更好地泛化到训练时未见过的序列长度，支持多分辨率生成。
- **并行注意力层**：在部分 block 中采用并行计算注意力和 MLP（而非串行），提升硬件利用效率。
- **QK 归一化**：对 Q 和 K 进行 RMSNorm，确保 bf16 混合精度训练的数值稳定性。

#### 6.3.5 Rectified Flow 的实现

FLUX 使用 Rectified Flow 作为训练框架：

- 训练时对每个样本在 t ~ U(0,1) 处采样，构造线性插值 x_t = (1-t) * noise + t * image_latent。
- 网络预测速度场 v_theta(x_t, t)，优化 || v_theta - (image_latent - noise) ||^2。
- 推理时从 noise ~ N(0,I) 出发，沿速度场做数值 ODE 积分。

**FLUX 变体的采样策略**：

| 变体 | 采样步数 | 机制 |
|:---|:---:|:---|
| FLUX.1 [pro] | ~25-50 步 | 完整 Rectified Flow 采样 |
| FLUX.1 [dev] | ~25-50 步 | Guidance distilled（无需 CFG） |
| FLUX.1 [schnell] | 1-4 步 | 对抗蒸馏 (Adversarial Distillation)，不需要 CFG |

**Guidance Distillation**：FLUX [dev] 通过蒸馏将 classifier-free guidance 的效果内化到模型中，使得推理时不需要同时运行条件和无条件前向传播，速度翻倍。模型接受一个 guidance 强度输入，通过专门的 guidance_in 嵌入层注入。

### 6.4 SD3 MMDiT：多模态 Diffusion Transformer

> 论文：*Scaling Rectified Flow Transformers for High-Resolution Image Synthesis* (Esser et al., 2024)

#### 6.4.1 MMDiT 的核心创新

SD3 的 Multimodal Diffusion Transformer (MMDiT) 是 DiT 向多模态方向的重要演进。其核心思想是：**文本和图像不应该通过简单的交叉注意力交互，而应该在同一个 Transformer 中以平等的地位进行联合注意力。**

**与传统交叉注意力的对比**：

```
传统 U-Net / DiT 的交叉注意力:
  图像 token → Q
  文本 token → K, V
  图像可以"查询"文本，但文本无法"查询"图像
  → 信息流是单向的

MMDiT 的联合注意力:
  图像 token → Q_img, K_img, V_img  (独立权重)
  文本 token → Q_txt, K_txt, V_txt  (独立权重)
  拼接后做联合注意力
  → 信息流是双向的：图像能看到文本，文本也能看到图像
```

这一设计的优势：
1. **文本理解更强**：文本 token 可以根据图像上下文动态调整自身表征。
2. **拼写准确性提升**：双向注意力让模型更好地理解文字与视觉的对应关系。
3. **模态平等**：两种模态不存在"主/从"关系，信息交互更充分。

#### 6.4.2 三文本编码器设计

SD3 使用三个文本编码器，各有分工：

| 编码器 | 参数量 | 作用 | 输出形式 |
|:---|:---:|:---|:---|
| CLIP ViT-L/14 | ~400M | 视觉-语义对齐（短文本概念） | pooled embedding + token 序列 |
| OpenCLIP ViT-bigG/14 | ~2.5B | 更丰富的视觉-语义表征 | pooled embedding + token 序列 |
| T5-XXL | ~4.7B | 深层语言理解（长文本、复杂语义） | token 序列 |

**使用方式**：
- 两个 CLIP 模型的 **pooled embedding**（全局向量）被拼接后，与时间步嵌入相加，用于 adaLN 调制——类似 DiT 的 AdaLN-Zero 机制。
- 三个编码器的 **token 序列** 被拼接为完整的文本 token 序列，作为 MMDiT 联合注意力的文本输入。

**推理时的灵活性**：
- 移除 T5-XXL（4.7B 参数）后，显存需求大幅下降，视觉美学质量几乎不受影响（对比胜率 50%），仅文字准确性略有下降（胜率 46%）。
- 这意味着资源受限时可以只用两个 CLIP 编码器进行推理。

#### 6.4.3 MMDiT Block 的具体结构

```
输入: 图像 token [x], 文本 token [c], 时间步+pooled 文本嵌入 [y]

y → MLP → (gamma_x, beta_x, alpha_x, gamma_c, beta_c, alpha_c)  ← 六组调制参数

图像分支:                              文本分支:
x → LN(gamma_x, beta_x)               c → LN(gamma_c, beta_c)
  → W_q^x, W_k^x, W_v^x                → W_q^c, W_k^c, W_v^c
       ↓                                     ↓
       └──→ Q = [Q_x; Q_c]  K = [K_x; K_c]  V = [V_x; V_c] ←──┘
                    ↓
              Joint Attention
                    ↓
              拆分为 attn_x, attn_c
       ↓                                     ↓
  × alpha_x + x (残差)                  × alpha_c + c (残差)
       ↓                                     ↓
  LN → MLP_x                            LN → MLP_c
       ↓                                     ↓
  × alpha'_x + (残差)                   × alpha'_c + (残差)
       ↓                                     ↓
  更新的图像 token                        更新的文本 token
```

**QKV 归一化**：SD3 对 Q 和 K 进行 RMSNorm 处理，这使得模型可以稳定地在 bf16 精度下训练，而不需要降低学习率或其他额外措施。

#### 6.4.4 SD3 vs FLUX 的架构对比

| 特性 | SD3 (MMDiT) | FLUX |
|:---|:---|:---|
| 参数量 | 2B (SD3 Medium) / 8B (SD3 Large) | 12B |
| 文本编码器 | 3 个 (CLIP-L + CLIP-bigG + T5) | 2 个 (CLIP-L + T5) |
| 注意力方式 | 全程联合注意力 | 前半段联合注意力 + 后半段统一序列 |
| 位置编码 | 正弦余弦 | RoPE |
| 训练框架 | Rectified Flow | Rectified Flow |
| 结构 | 纯 MMDiT blocks | Double-Stream + Single-Stream blocks |

FLUX 可以视为 MMDiT 的演进：先用 Double-Stream 做模态专属处理（类似 MMDiT），再用 Single-Stream 做深度融合（更激进的融合方式）。

### 6.5 自回归图像生成 (Autoregressive Image Generation)

> 核心论文：
> - *Neural Discrete Representation Learning* (VQ-VAE, van den Oord et al., NeurIPS 2017)
> - *Taming Transformers for High-Resolution Image Synthesis* (VQ-GAN, Esser et al., CVPR 2021)
> - *Visual Autoregressive Modeling: Scalable Image Generation via Next-Scale Prediction* (VAR, NeurIPS 2024 Best Paper)
> - *Transfusion: Predict the Next Token and Diffuse Images with One Multi-Modal Model* (Meta, 2024)

#### 6.5.1 基础：VQ-VAE 与图像 token 化

自回归图像生成的前提是将连续的图像变为离散的 token 序列，这样才能使用 LLM 的 next-token prediction 范式。

**VQ-VAE 的工作流程**：

```
输入图像 x (256x256x3)
    ↓ Encoder (CNN)
连续潜表示 z_e (16x16x256)    ← 每个空间位置是一个 256 维向量
    ↓ Vector Quantization
离散 token 序列 (16x16 = 256 个 token)
    ↓ 查找 Codebook
量化后的表示 z_q (16x16x256)  ← 每个向量被替换为 codebook 中最近的向量
    ↓ Decoder (CNN)
重建图像 x' (256x256x3)
```

**向量量化的关键**：
1. **Codebook（码本）**：维护一个大小为 K 的嵌入字典 {e_1, e_2, ..., e_K}，每个 e_i 是一个 d 维向量。典型设置 K=8192, d=256。
2. **量化过程**：对编码器输出的每个向量 z_e，找到 codebook 中距离最近的向量：`k = argmin_j ||z_e - e_j||^2`，然后用 e_k 替代 z_e。
3. **直通估计器 (Straight-Through Estimator)**：量化操作不可导，因此反向传播时直接将梯度从 z_q 复制到 z_e，绕过量化步骤。
4. **Commitment Loss（承诺损失）**：`||z_e - sg[e_k]||^2`，鼓励编码器输出靠近所选的 codebook 向量，防止编码器输出"飘移"。

**VQ-GAN 的改进**：
- 在 VQ-VAE 基础上加入 GAN 判别器损失和感知损失 (perceptual loss)，大幅提升重建质量。
- 在高分辨率图像合成中效果显著优于纯 VQ-VAE。
- 成为后续所有自回归图像生成的标准 tokenizer。

#### 6.5.2 传统自回归方式：光栅扫描

传统方法（如 DALL-E 1、Parti、LlamaGen）将量化后的 2D token map 展平为 1D 序列，按光栅扫描顺序 (左→右, 上→下) 逐 token 生成：

```
[t_1, t_2, ..., t_256]  (16x16 的 token map 展平)

生成过程:
  p(t_1) → p(t_2 | t_1) → p(t_3 | t_1, t_2) → ... → p(t_256 | t_1,...,t_255)
```

**问题**：
- 序列很长（256x256 图像可能有数千个 token），推理慢。
- 光栅扫描顺序不符合人类创作图像的逻辑（先画全局轮廓，再画细节）。
- 每个 token 只能看到已生成的 token（因果注意力），全局一致性差。

#### 6.5.3 VAR：下一尺度预测 (Next-Scale Prediction)

VAR (Visual Autoregressive Modeling) 是 NeurIPS 2024 最佳论文，提出了全新的自回归范式：

**核心思想**：不是逐 token 生成，而是逐**分辨率/尺度**生成——从粗到细。

```
传统 AR:  [t_1] → [t_2] → [t_3] → ... → [t_n]     (逐 token)

VAR:      [1x1] → [2x2] → [4x4] → [8x8] → ... → [16x16]  (逐尺度)
          全局    粗糙     中等     精细            最终
```

**多尺度 VQ-VAE**：

VAR 设计了一种特殊的 VQ-VAE，将图像编码为多尺度 token map：

```
图像 → Encoder → 多尺度 token maps:
  Scale 1: 1×1   (1 个 token，全局信息)
  Scale 2: 2×2   (4 个 token)
  Scale 3: 4×4   (16 个 token)
  Scale 4: 8×8   (64 个 token)
  ...
  Scale K: 16×16 (256 个 token，细节信息)
```

**生成过程**：
1. 先生成 1x1 的全局 token（类似"画一个大概轮廓"）。
2. 基于已有信息，**并行**生成下一尺度 2x2 的所有 4 个 token。
3. 继续并行生成 4x4、8x8... 直到最终分辨率。
4. 每个尺度内部的 token 是并行生成的，仅尺度之间是自回归的。

**性能突破**：

| 模型 | FID ↓ | IS ↑ | 推理速度 |
|:---|:---:|:---:|:---|
| 传统 AR (LlamaGen) | 18.65 | 80.4 | 慢（逐 token） |
| **VAR** | **1.73** | **350.2** | **~20x 更快** |
| DiT-XL/2 (扩散) | 2.27 | 278.2 | 中等 |

VAR 首次证明了 GPT 风格的自回归模型可以超越扩散 Transformer。并且 VAR 展现出清晰的幂律 Scaling Laws（线性相关系数 -0.998），与 LLM 的规律惊人一致。

#### 6.5.4 GPT-4o 的原生图像生成

GPT-4o (2025.03) 实现了真正的"统一多模态模型"——同一个 Transformer 既生成文本又生成图像。

**推测的技术架构**（OpenAI 未公开论文）：

```
统一 Transformer 骨干
├── 文本 token → 自回归 next-token prediction (标准 LLM 方式)
└── 图像 token → 可能采用连续表征 + 分组扩散解码

输入:  [文本 token_1, ..., 文本 token_n, <IMG_START>, 图像 token_1, ...]
输出:  逐 token 生成（文本和图像交错）
```

**关键技术推测**：

1. **图像 tokenizer**：可能使用连续 token（而非离散 VQ），避免量化造成的信息瓶颈。连续 token 允许更平滑的过渡和更自然的视觉效果。
2. **分层生成**：可能类似 VAR 的 next-scale prediction，先生成低分辨率框架，逐步细化。
3. **分组扩散解码**：视觉 token 的解码可能采用分组扩散过程，逐批（而非一次性）对图像区域去噪。
4. **统一训练**：文本和图像在同一序列中交错训练，使得模型天然理解两种模态的关系。

**GPT-4o 图像生成的独特优势**：

- **指令遵循**：因为是 LLM，对自然语言指令的理解远超扩散模型的文本编码器。
- **文字渲染**：自回归模型天然适合生成文字（与生成文本 token 的机制一致）。
- **多轮编辑**：可以在对话中自然地修改已生成的图像（"把背景改成蓝色"）。
- **上下文理解**：可以理解上传的图像并基于理解进行编辑或创作。

#### 6.5.5 Transfusion：混合自回归与扩散

Transfusion (Meta, 2024) 提出了另一种优雅的统一方案：

**核心思想**：不同模态使用不同的损失函数，但共享一个 Transformer 的参数。

```
混合序列: [text_1, text_2, ..., <IMG>, patch_1, patch_2, ..., patch_16, </IMG>, text_k, ...]

文本部分: 使用标准 next-token prediction loss (交叉熵)
图像部分: 使用扩散 loss (预测噪声 / 速度)
```

**架构设计**：

1. **图像表示**：使用 VAE 将图像编码为连续的 latent patches（如 16 个 patch），不做离散量化。相比 VQ 方法避免了信息瓶颈。
2. **序列构造**：文本 token 和图像 patch 交错排列成一个序列。
3. **混合注意力模式**：
   - 所有元素之间使用因果注意力（文本的标准方式）。
   - 同一图像内部的 patch 之间额外使用双向注意力（图像 patch 可以互相看到同一图像的所有 patch）。
4. **模态专属层**：Transformer 共享绝大部分参数，但编码层和解码层是模态专属的（文本用 embedding + LM head，图像用 VAE encoder + diffusion head）。

**训练细节**：
- 在 50% 文本 + 50% 图像数据上联合预训练。
- 文本部分用 language modeling loss，图像部分用 diffusion loss。
- 两个 loss 同时反向传播，更新共享的 Transformer 参数。

**Transfusion vs 离散 token 化 (如 Chameleon)**：

| 特性 | Chameleon (离散 token) | Transfusion (连续 + 扩散) |
|:---|:---|:---|
| 图像表示 | VQ 离散 token | 连续 latent patches |
| 图像 loss | 交叉熵 (next-token) | 扩散损失 |
| 信息瓶颈 | 有（量化损失） | 无 |
| 扩展性 (Scaling) | 较差 | **显著更好** |
| 图像质量 | 受限于 codebook 大小 | 无量化上界 |

Transfusion 在论文中展示了远优于离散方法的 scaling 曲线，证明连续表征 + 扩散损失是处理图像模态的更好选择。

#### 6.5.6 自回归图像生成的未来方向

1. **统一多模态模型**：GPT-4o 和 Transfusion 证明了文本-图像统一生成的可行性，这可能成为未来多模态 AI 的标准范式。
2. **连续 vs 离散之争**：离散 token 简单但有信息瓶颈；连续表征质量更高但需要扩散解码。混合方案（如 Transfusion）可能是最优解。
3. **更好的视觉 tokenizer**：VQ-GAN 的重建质量仍是自回归方法的瓶颈，更先进的 tokenizer（更大 codebook、residual quantization、lookup-free quantization）持续推进中。
4. **Scaling Laws 的统一**：VAR 和 DiT 都展现了视觉生成的 Scaling Laws。随着统一模型（文本 + 图像 + 视频 + 音频）的出现，跨模态的统一 Scaling Laws 将成为重要研究方向。
