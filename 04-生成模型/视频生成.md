# 视频生成

> 2025-2026 是 AI 视频生成从技术演示走向实际应用的转折点。

## 一、技术架构

### 核心组件
```
视频生成模型
├── 3D VAE (视频编解码器)
│   将视频压缩到潜空间
│   典型压缩：4x 时间 × 8x8 空间
│
├── DiT (Diffusion Transformer) 骨干
│   在潜空间做时空去噪
│   Full 3D Attention (空间 + 时间)
│
├── Flow Matching 训练
│   Rectified Flow → 更少采样步骤
│
└── 文本编码器
    T5-XXL / CLIP 提供文本条件
```

### 3D VAE (Causal Video VAE)
- 将视频在空间和时间维度同时压缩
- **因果设计**：每帧只能关注之前的帧（适合自回归生成）
- 所有主流模型都使用：HunyuanVideo, CogVideoX, Wan, Kling

### 3D Full Attention
- 在空间和时间维度上做完整的注意力
- 保证最好的时间一致性
- 计算代价高，但质量最好
- 替代方案：分离的空间注意力 + 时间注意力（质量较差但更快）

## 七、3D VAE 视频编解码器深度解析

### 7.1 为什么需要 3D VAE

视频生成直接在像素空间操作的计算代价极其高昂。以一段 720p、5 秒、24fps 的视频为例，原始数据量为 `120帧 x 720 x 1280 x 3通道 ≈ 3.3亿像素`。如果 Diffusion Transformer 直接在像素空间去噪，所需的注意力计算量 (`O(n^2)`) 将完全不可行。

3D VAE（三维变分自编码器）的核心作用是将视频压缩到一个低维的**潜空间（latent space）**，使得后续的 DiT 去噪过程在计算上变得可行。

### 7.2 时空联合压缩

与图像 VAE（如 Stable Diffusion 使用的 2D VAE，仅做空间压缩 `8x8`）不同，3D VAE 同时在**时间维度**和**空间维度**进行压缩：

```
输入视频: [T帧, H高, W宽, 3通道]
    ↓ 3D VAE Encoder
潜空间表示: [T/ct, H/cs, W/cs, C通道]
    ↓ DiT 去噪（在潜空间完成）
    ↓ 3D VAE Decoder
输出视频: [T帧, H高, W宽, 3通道]
```

典型的压缩比为 **4x8x8**（时间4倍、空间各8倍），这意味着：
- 时间维度：每4帧压缩成1个潜空间帧
- 空间维度：高和宽各缩小8倍
- 通道维度：从3个RGB通道扩展到 `C` 个潜空间通道（通常为 16 或 32）

**实际压缩效果示例**（HunyuanVideo）：
```
原始视频: 129帧 x 720 x 1280 x 3 → 约 3.5亿个值
潜空间:   33帧 x 90 x 160 x 16  → 约 760万个值
总压缩倍率: ≈ 46倍（元素数）/ 185倍（考虑通道变化后）
```

### 7.3 因果设计（Causal Design）

3D VAE 中的"因果"指的是**时间因果卷积（Temporal Causal Convolution）**：

- **核心原则**：在编码第 `t` 帧时，卷积核只能看到第 `t` 帧及其之前的帧，不能看到未来的帧
- **实现方式**：在时间维度上使用单向填充（padding），使得卷积核的感受野只覆盖过去
- **对比**：普通 3D 卷积会在时间维度使用对称填充，能看到前后帧

```
普通 3D 卷积（非因果）:
  帧 t-2, t-1, [t], t+1, t+2 → 输出 t
  问题：依赖未来帧，不适合流式/自回归生成

因果 3D 卷积:
  帧 t-4, t-3, t-2, t-1, [t] → 输出 t
  优势：只依赖历史帧，支持逐帧/逐段生成
```

**因果设计的优势**：
1. **兼容图像和视频**：单帧输入时（图像），因果卷积退化为普通 2D 卷积，无需为图像和视频维护两套 VAE
2. **支持流式编码**：可以边接收帧边编码，不需要等整段视频到齐
3. **无限长度视频处理**：Wan-VAE 通过因果卷积配合**特征缓存机制（feature cache）**，能编解码任意长度的 1080P 视频，无需将完整视频加载到内存
4. **自回归生成兼容**：与自回归视频生成范式天然契合

### 7.4 各模型 3D VAE 对比

| 模型 | 压缩比 (TxHxW) | 潜通道数 | 特殊设计 |
|------|----------------|---------|----------|
| CogVideoX | 4x8x8 | 16 | 时间因果卷积 + 上下文并行 |
| HunyuanVideo | 4x8x8 | 16 | CausalConv3D，兼容图像/视频 |
| Wan-VAE (Wan 2.1/2.2) | 4x8x8 | 16 | U-Net 架构 + RMSNorm + 特征缓存，支持无限长度 1080P |
| Cosmos Tokenizer | 8x16x16 | - | 连续/离散两种 token 模式，压缩率最高 |
| Open-Sora VAE | 4x8x8 | 4 | 基于 SD 2D VAE 扩展时间维度 |

### 7.5 训练目标

3D VAE 的训练通常结合多个损失函数：

```
总损失 = λ_rec * L_reconstruction + λ_kl * L_KL + λ_perc * L_perceptual + λ_gan * L_adversarial
```

1. **重建损失（Reconstruction Loss）**：L1 或 L2 距离，确保解码后的视频与输入尽量一致
2. **KL 散度（KL Divergence）**：约束潜空间分布接近标准正态分布 `N(0,1)`，使潜空间平滑、可采样
3. **感知损失（Perceptual Loss / LPIPS）**：使用预训练 VGG 网络提取特征，在特征层面衡量相似度，确保感知质量而非仅像素匹配
4. **对抗损失（Adversarial / GAN Loss）**：使用 PatchGAN 判别器，使解码输出在局部细节上更加真实锐利

训练策略通常是**先图像后视频**的渐进方式：
- 阶段一：用大量图像数据训练 2D 压缩能力
- 阶段二：在视频数据上微调，学习时间维度的压缩与重建
- 这样做的好处是充分利用丰富的图像数据建立空间压缩基础

## 八、Sora 架构深度解析

### 8.1 技术报告核心思想

OpenAI 在 2024 年 2 月发布了技术报告《Video generation models as world simulators》，这份报告虽然没有披露完整的架构细节，但提出了视频生成领域几个关键的架构创新思想。

**核心命题**：通过在大规模视频数据上训练通用的视频生成模型，可以构建物理世界的通用模拟器（general purpose simulators of the physical world）。

### 8.2 时空 Patch（Spacetime Patches）

Sora 最重要的架构创新是**时空 Patch**，它将 Vision Transformer（ViT）中的 Patch 概念扩展到了视频领域：

```
处理流程:
1. 原始视频 → 视觉编码器 → 低维潜空间表示（时间+空间压缩）
2. 潜空间表示 → 分解为时空 Patch 序列
3. 时空 Patch 序列 → 作为 Transformer token 输入 DiT
```

**时空 Patch 的本质**：每个 Patch 是一个小的**时空立方体**（spacetime cube），包含若干帧在某个空间区域的信息。它不是单帧的图像块，而是跨越时间和空间的三维信息单元。

**灵感来源**：Sora 的技术报告引用了 NaViT（*Patch n' Pack: NaViT, a Vision Transformer for any Aspect Ratio and Resolution*，Dehghani et al., 2023），这是 Google 提出的支持任意分辨率和宽高比的 ViT 架构。

### 8.3 可变分辨率/时长/宽高比

传统视频生成模型通常将所有视频裁剪或缩放到固定尺寸进行训练，这会导致构图不自然、主体被裁切等问题。Sora 的 Patch 化设计使其能够在**原始尺寸**上训练：

```
传统方法:
  所有视频 → 裁剪/缩放到 256x256 → 训练
  问题：损失构图信息，横竖屏视频被强制裁切

Sora 的方法:
  1920x1080 横屏视频 → 提取对应 grid 大小的 Patch 序列
  1080x1920 竖屏视频 → 提取不同 grid 大小的 Patch 序列
  480p 短视频 → 更少的 Patch
  4K 长视频 → 更多的 Patch
```

**在原始分辨率上训练的好处**：
1. **更好的构图**：模型学到自然的视觉构图规律
2. **灵活的输出**：推理时可以通过调整 Patch 网格大小来生成任意尺寸的视频
3. **统一训练**：图像（单帧视频）和视频可以在同一框架下训练

### 8.4 Diffusion Transformer 骨干

Sora 使用 Diffusion Transformer（DiT）作为骨干网络：
- 输入：带噪声的时空 Patch 序列 + 文本条件
- 训练目标：预测原始的"干净" Patch（去噪）
- 解码：训练一个对应的解码器将生成的潜空间表示映射回像素空间

### 8.5 Scaling 特性与涌现能力

技术报告中展示了一些令人瞩目的 Scaling 特性：

1. **视频质量随计算量提升**：使用更多计算训练的模型生成更高质量的视频
2. **3D 一致性涌现**：模型能生成动态的 3D 场景，即使没有显式的 3D 监督
3. **长程连贯性**：能够在长视频中维持角色和场景的一致性
4. **世界交互模拟**：展现出对简单物理规律的理解（如物体碰撞、液体流动）

### 8.6 Sora 2 的关键升级（2025.09）

Sora 2 在初代基础上进一步提升：
- **时长扩展**：最长 25 秒单次生成（行业最长）
- **原生音频**：同步生成音效、环境音、对白
- **多角色交互**：复杂的角色动态和多元素场景编排
- **叙事能力**：长时间段的叙事连贯性

## 九、HunyuanVideo 架构深度解析

### 9.1 整体定位

HunyuanVideo（腾讯混元视频）是 2024 年 12 月发布的**最大规模开源视频生成模型**，拥有超过 130 亿参数。它在发布时的效果超越了所有开源模型，甚至在多项评测中接近闭源的顶尖模型。论文标题为《HunyuanVideo: A Systematic Framework For Large Video Generative Models》。

### 9.2 Dual-Stream to Single-Stream DiT 设计

这是 HunyuanVideo 最核心的架构创新，直接借鉴了 Black Forest Labs 的 **FLUX** 模型在图像生成领域验证的双流到单流设计：

```
文本 Token 序列 ──┐
                  ├─→ [Dual-Stream Blocks] ─→ [Single-Stream Blocks] ─→ 输出
视频 Patch 序列 ──┘

Dual-Stream Block (MM-DiT 风格):
┌──────────────────────────────────┐
│  文本分支:                        │
│  LayerNorm → Self-Attention → FFN │
│                                   │
│  视频分支:                        │
│  LayerNorm → Self-Attention → FFN │
│                                   │
│  交叉：Q/K/V 在注意力层共享计算   │
└──────────────────────────────────┘

Single-Stream Block:
┌──────────────────────────────────┐
│  [文本 Token ⊕ 视频 Token] 拼接  │
│  → LayerNorm → Self-Attention    │
│  → FFN → 输出                    │
└──────────────────────────────────┘
```

**双流阶段（Dual-Stream）**：
- 文本和视频 Token 各自独立通过多个 Transformer Block
- 每个模态拥有独立的调制参数（modulation），避免相互干扰
- 让每个模态在自己的空间内学习最优的表征
- 对应 FLUX 中的 MM-DiT（Multi-Modal DiT）设计

**单流阶段（Single-Stream）**：
- 将文本和视频 Token 拼接后送入后续的 Transformer Block
- 实现深层次的多模态信息融合
- 捕获视觉和语义之间的复杂交互关系
- 计算效率更高（一次注意力同时处理两种模态）

**与 FLUX 的关系**：
- FLUX 在图像生成中首先验证了 Dual-Stream → Single-Stream 的有效性
- HunyuanVideo 将这一设计从 2D 图像扩展到了 3D 视频
- 核心差异在于 HunyuanVideo 的注意力需要处理时空两个维度

### 9.3 文本编码器：MLLM 替代 T5

HunyuanVideo 在文本编码器上做了重要创新——使用**多模态大语言模型（MLLM）**替代传统的 T5-XXL：

| 特性 | T5-XXL (传统) | MLLM Decoder-Only (HunyuanVideo) |
|------|-------------|----------------------------------|
| 架构 | Encoder-Decoder | Decoder-Only |
| 图文对齐 | 较弱 | 经过视觉指令微调，更强 |
| 细节描述 | 一般 | 更擅长图像细节描述 |
| 复杂推理 | 有限 | 更强的推理能力 |
| 参数规模 | 11B | 通常 7B+ |

同时使用 CLIP-Large 的最终 token 特征作为**全局引导信号**，注入到 DiT Block 中。

### 9.4 3D VAE 与 Flow Matching

- **3D VAE**：CausalConv3D，压缩比 4x8x8，潜空间通道 16
- **训练范式**：Flow Matching（流匹配），相比传统 DDPM 具有更直的采样轨迹，允许更少的采样步数达到相似质量
- **生成能力**：720p 分辨率、5 秒视频（129 帧）

### 9.5 HunyuanVideo 1.5 改进

2025 年发布的 1.5 版本进一步优化：
- **轻量化**：8.3B 参数版本，可在消费级 GPU 上运行
- **更长视频**：扩展了生成时长
- **图生视频**：增加了 Image-to-Video 能力

## 十、Wan 2.1 / 2.2（万相）架构深度解析

### 10.1 Wan 系列概述

Wan（万相）是阿里巴巴通义团队开发的视频生成模型系列，**完全开源**（代码 + 权重），在开源社区产生了巨大影响。Wan 2.1 建立了开源视频生成的新标杆，Wan 2.2 则是全球首个采用 MoE 架构的视频生成扩散模型。

### 10.2 Wan-VAE：支持无限长度的 3D 因果 VAE

Wan-VAE 是 Wan 系列的核心组件之一，具有独特的设计：

```
架构: U-Net + 3D Causal Convolution
压缩比: 4x8x8 (时间x空间x空间)
归一化: RMSNorm (替代 GroupNorm)
特殊能力: 编解码任意长度的 1080P 视频

关键创新 —— 特征缓存机制:
┌──────────────────────────────────────┐
│ 传统方式: 整段视频 → 一次性编码       │
│ 问题: 长视频会 OOM                    │
│                                       │
│ Wan-VAE: 分段编码 + 特征缓存          │
│ 段1 → 编码 → 缓存边界特征            │
│ 段2 → 编码 (利用段1缓存) → 缓存      │
│ 段3 → 编码 (利用段2缓存) → ...       │
│ 优势: 内存恒定，长度无限              │
└──────────────────────────────────────┘
```

使用 RMSNorm 替代传统的 GroupNorm，在保持因果性的同时提升了训练稳定性和推理效率。

### 10.3 Wan 2.1 基础架构

Wan 2.1 的核心是一个标准的 DiT（Diffusion Transformer），配合 Flow Matching 训练：

- **文本编码器**：UMT5（多语言 T5 变体），天然支持中英文等多种语言
- **条件注入**：Cross-Attention 机制嵌入每个 Transformer Block
- **时间步调制**：通过 MLP 生成每个时间步的专用调制参数，控制去噪过程
- **模型规格**：
  - 14B：最高质量，720p
  - 1.3B：轻量版，适合资源受限场景

### 10.4 Wan 2.2 MoE 扩散架构（核心创新）

Wan 2.2 的最大突破是将 **Mixture-of-Experts (MoE)** 架构引入视频扩散模型。这是一个极具洞察力的设计：

**核心观察**：扩散模型在不同的去噪阶段需要完成根本不同的任务——
- **高噪声阶段**（早期步骤）：确定整体布局、构图、大色块
- **低噪声阶段**（后期步骤）：精细化纹理、边缘、细节

**MoE 解决方案**：

```
Wan 2.2 A14B 模型结构:
┌───────────────────────────────────────┐
│  总参数量: 27B                         │
│  活跃参数量: 14B (每步只用一个专家)     │
│                                        │
│  Expert 1 (高噪声专家 ~14B):           │
│  → 专注整体布局和结构                   │
│  → 在去噪早期阶段激活                   │
│                                        │
│  Expert 2 (低噪声专家 ~14B):           │
│  → 专注细节纹理和精细化                 │
│  → 在去噪后期阶段激活                   │
│                                        │
│  路由机制: 基于当前噪声水平选择专家       │
└───────────────────────────────────────┘
```

**MoE 带来的优势**：
1. **计算不增加**：虽然总参数翻倍（27B），但每步推理只激活 14B，计算量和显存与 Wan 2.1 的 14B 模型几乎相同
2. **质量提升**：每个专家专精于自己负责的去噪阶段，比单一模型处理所有阶段效果更好
3. **隐式任务分工**：高噪声专家学习"画大画"，低噪声专家学习"精修"

### 10.5 训练数据与开源生态

Wan 2.2 相比 Wan 2.1 大幅扩充了训练数据：
- **图像数据增加 65.6%**
- **视频数据增加 83.2%**
- 精心策划的美学数据，包含光影、构图、对比度、色调等详细标签
- 支持可定制的电影风格生成

**开源版本矩阵**：

| 模型 | 类型 | 参数 | 特点 |
|------|------|------|------|
| Wan 2.2-T2V-A14B | 文生视频 | 27B (14B active) | MoE，最高质量 |
| Wan 2.2-I2V-A14B | 图生视频 | 27B (14B active) | MoE，图片动画 |
| Wan 2.2-IT2V-5B | 统一生成 | 5B | 轻量，图文均可 |

## 十一、世界模型（World Models）深度解析

### 11.1 从视频生成到世界模拟

"世界模型"代表了视频生成技术的范式转变——不再把目标定义为"生成好看的视频"，而是"学习物理世界的运行规律"。核心区别在于：

```
传统视频生成:
  输入: 文本描述 → 输出: 好看的视频
  评价标准: 视觉质量、文本匹配度

世界模型:
  输入: 状态 + 动作 → 输出: 下一个状态（可交互）
  评价标准: 物理准确性、一致性、可交互性
  应用: 机器人训练、自动驾驶仿真、游戏
```

### 11.2 NVIDIA Cosmos

Cosmos 是 NVIDIA 在 2025 年 1 月发布的**物理 AI 世界基础模型平台**，专为机器人和自动驾驶等物理 AI 应用设计。

**平台组成**：

```
Cosmos 平台
├── Cosmos Tokenizer (视频编解码)
│   ├── 连续 Tokenizer → 输出潜空间向量 (用于扩散模型)
│   ├── 离散 Tokenizer → 输出整数 Token (用于自回归模型)
│   ├── 图像 Tokenizer: 空间压缩
│   └── 视频 Tokenizer: 时空压缩 (8x16x16)
│
├── Cosmos-Predict1 (自回归世界模型)
│   ├── 架构: Transformer Decoder
│   ├── 输入: 文本 + 过去视频帧 → 预测未来帧
│   ├── 渐进式训练: 17帧 → 34帧 → 121帧 (50000 tokens)
│   ├── FPS-aware 3D RoPE: 位置编码感知帧率
│   └── 文本条件: T5-XXL Cross-Attention
│
└── Cosmos-Predict2 (扩散世界模型)
    ├── 架构: Transformer + 扩散去噪
    ├── 输入: 文本/视频 → 生成未来世界状态
    └── 应用: Text2World, Video2World
```

**Cosmos Tokenizer 的技术特色**：
- 压缩比高达 **8x16x16**（时间8倍、空间各16倍），是同类最高
- 同时提供**连续 token**（潜空间向量，供扩散模型使用）和**离散 token**（整数索引，供自回归模型使用）
- 处理速度比同期最佳 tokenizer 快 12 倍
- 总压缩率高 8 倍

**训练数据规模**：在 9000 万亿 token 上预训练，包含 2000 万小时的自动驾驶、机器人、合成环境等数据。

### 11.3 Genie 3（Google DeepMind）

Genie 3 是 Google DeepMind 开发的**通用世界模型**，能够从文本提示生成可实时交互的 3D 环境。这是世界模型概念最直观的体现——你可以"走进"生成的世界。

**三大核心组件**：

```
1. 时空视频 Tokenizer (VQ-VAE)
   视频帧 → 离散视觉 Token
   低比特率压缩，支持 24fps 实时处理

2. 自回归动力学模型 (Dynamics Model)
   架构: Decoder-only 时空 Transformer
   训练: Teacher-forcing 预测下一帧 Token
   读取所有过去 Token + 动作 → 生成下一帧

3. 潜在动作模型 (Latent Action Model)
   学习 8 维潜在动作 Token
   从视频中自动发现动作语义（移动、转向等）
   无需人工动作标注
```

**关键能力**：
- **实时交互**：720p 分辨率、24fps 实时生成
- **视觉记忆**：环境一致性可维持数分钟，视觉记忆可追溯到约一分钟前
- **涌现一致性**：3D 一致性不是通过 NeRF 等显式 3D 建模实现的，而是大规模训练后自发涌现的能力
- **回访一致性**：用户回到之前到过的位置时，模型会参考一分钟前的信息来保持场景一致

**技术挑战**：在自回归生成每一帧时，模型需要参考不断增长的历史轨迹。如果用户在一分钟后回到某个位置，模型需要回溯一分钟前的相关信息，而这些计算必须在每秒内完成多次。

### 11.4 世界模型 vs 视频生成模型对比

| 维度 | 视频生成模型 | 世界模型 |
|------|-------------|---------|
| 输出 | 固定视频片段 | 可交互的连续环境 |
| 物理理解 | 隐式学习，不保证 | 显式目标，物理准确 |
| 用户控制 | 文本/图像条件 | 实时动作输入 |
| 时间范围 | 5-25 秒 | 理论上无限 |
| 一致性 | 片段内一致 | 长期环境持久性 |
| 应用领域 | 内容创作 | 机器人、自动驾驶、游戏 |

## 十二、视频生成关键训练技术

### 12.1 渐进式训练（Progressive Training）

渐进式训练是当前所有 SOTA 视频生成模型共同采用的核心训练策略。其基本思想是**从简单到复杂**，分阶段逐步提升模型能力：

```
典型的渐进式训练流程:

阶段 0: 图像预训练
  └── 在大规模图像数据上训练 DiT 的空间生成能力
  └── 目标: 学习静态视觉内容的生成

阶段 1: 低分辨率短视频
  └── 256x256, 16帧
  └── 目标: 学习基本的时间动态

阶段 2: 中等分辨率 + 更长时间
  └── 512x512, 32-48帧
  └── 目标: 学习更复杂的运动模式

阶段 3: 高分辨率长视频
  └── 720p/1080p, 100+帧
  └── 目标: 高质量、长时间的视频生成

阶段 4: 质量微调
  └── 在精心筛选的高质量数据上微调
  └── 目标: 美学质量和物理准确性
```

**各模型的渐进式训练实践**：

- **CogVideoX**：使用渐进式训练 + 多分辨率帧打包技术，逐步从短视频扩展到 6 秒连贯视频
- **Cosmos-Predict1**：从预测 17 帧开始，扩展到 34 帧，最终达到 121 帧（50000 tokens）
- **Open-Sora**：先用图像预训练权重初始化，然后稀疏注意力训练，再联合训练图像和不同分辨率/时长的视频

### 12.2 Flow Matching（流匹配）训练

Flow Matching 已经取代传统的 DDPM/DDIM 成为视频生成的主流训练范式：

```
传统 DDPM:
  前向过程: x_0 → 加噪 → x_t (马尔可夫链，多步)
  训练目标: 预测噪声 ε
  采样: 需要 50-1000 步去噪
  轨迹: 曲折的随机路径

Flow Matching (Rectified Flow):
  前向过程: x_0 → x_t = (1-t)·x_0 + t·ε (线性插值)
  训练目标: 预测速度向量 v = ε - x_0
  采样: 可以少至 10-30 步
  轨迹: 近似直线路径
```

**核心优势**：
1. **直线轨迹**：从噪声到数据的路径近似直线，减少采样步数
2. **速度预测**：学习"数据应该往哪个方向移动"，比预测噪声更直观
3. **确定性采样**：ODE 求解器，结果更稳定
4. **训练简单**：损失函数是预测速度与真实速度的 MSE

**使用 Flow Matching 的主流模型**：HunyuanVideo、Wan 2.1/2.2、FLUX、Stable Diffusion 3

### 12.3 时间一致性保障机制

时间一致性（temporal consistency）是视频生成区别于图像生成的核心挑战。主要解决方案包括：

**方案一：3D Full Attention**
```
最直接但计算最昂贵的方案
每个时空位置可以关注所有其他时空位置
[帧1-Patch1, 帧1-Patch2, ..., 帧T-PatchN] 全连接

优点: 时间一致性最好
缺点: 计算量 O((T*H*W)^2)，随帧数急剧增长
使用者: HunyuanVideo, Wan, Sora (推测)
```

**方案二：分离的时空注意力**
```
先做空间注意力，再做时间注意力（或反过来）
空间 Attention: 每帧内部的 Patch 之间
时间 Attention: 同一空间位置跨帧

优点: 计算量显著降低 O(T*(H*W)^2 + H*W*T^2)
缺点: 无法捕获某些复杂的时空交互
使用者: 早期模型，部分轻量模型
```

**方案三：因果注意力 + 自回归**
```
逐帧或逐段生成，每次只关注之前的内容
适合超长视频和实时交互场景

优点: 支持无限长度生成
缺点: 误差累积，后期帧质量可能下降
使用者: Genie 3, 部分长视频模型
```

**方案四：Lumiere 的全时长单次生成**
```
STUNet (Space-Time U-Net) 在时间和空间同时下采样
一次性生成完整时长的视频（非自回归）

优点: 全局时间一致，无分段拼接痕迹
缺点: 时长受限于显存
使用者: Lumiere (Google)
```

### 12.4 Classifier-Free Guidance (CFG) 与 CFG 蒸馏

CFG 是所有条件生成模型提升文本对齐度的关键技术：

```
标准 CFG 推理:
  output = uncond_output + guidance_scale * (cond_output - uncond_output)
  问题: 每步需要两次前向传播（有条件 + 无条件），推理速度减半

CFG 蒸馏:
  训练一个学生模型，使其单次前向传播就能产生类似 CFG 的效果
  推理速度翻倍
  HunyuanVideo 提供了 CFG 蒸馏版本 (cfgdistill)
```

### 12.5 多分辨率帧打包（Multi-Resolution Frame Packing）

CogVideoX 提出的技术，将不同分辨率和时长的视频样本打包到同一个训练 Batch 中：

```
传统方式:
  Batch 内所有样本必须相同尺寸 → 大量裁剪/填充 → 浪费计算

帧打包:
  样本A: 480p, 32帧 → token 数 = N_a
  样本B: 720p, 16帧 → token 数 = N_b
  样本C: 256p, 64帧 → token 数 = N_c
  打包: [样本A tokens | 样本B tokens | 样本C tokens]
  使用注意力掩码确保样本之间不交互

优势:
  - GPU 利用率更高
  - 自然支持多分辨率/多时长训练
  - 不需要裁剪或填充
```

### 12.6 CogVideoX Expert Transformer

CogVideoX 在 DiT 基础上提出了 **Expert Adaptive LayerNorm** 来改善文本-视频对齐：

```
标准 DiT:
  文本和视频使用相同的 LayerNorm 参数
  问题: 两种模态的分布差异大，共享归一化不够灵活

Expert Transformer (CogVideoX):
  文本 Token → Expert LayerNorm (文本专用参数)
  视频 Token → Expert LayerNorm (视频专用参数)
  Attention 层: 文本和视频 Token 拼接后联合计算

  效果: 每种模态有适合自己分布的归一化，
        同时在注意力层实现深度融合
```

这与 Wan 2.2 的 MoE 思想有异曲同工之处——都是在模型中引入"专业化"组件来处理不同类型的输入或不同阶段的任务。
