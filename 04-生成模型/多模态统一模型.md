# 多模态统一模型

> 从 "理解" 和 "生成" 分离，到一个模型统一完成两者——这是 2025 年最重要的架构趋势之一。

## 一、动机

### 为什么需要统一？
```
传统方案：
  理解：VLM (视觉语言模型) → 看图回答问题
  生成：扩散模型 → 根据文字生成图片
  → 两个独立模型，无法自然地 "看了之后画"

统一方案：
  一个模型：看图 → 推理 → 回答 → 画图 → 编辑
  → 自然的多模态交互
  → GPT-4o 就是这个方向
```

## 二、技术路线

### 路线 1：统一自回归 (Unified AR) ★ 主流趋势

```
核心：所有模态 (文本/图像/音频/视频) 都 token 化
       → 用一个自回归 Transformer 统一生成

文本 token: 标准 tokenizer (BPE)
图像 token: VQ-VAE / VQ-GAN → 离散 token
音频 token: 音频编解码器 → 离散 token

生成方式: [文本 tokens] → [图像 tokens] → [文本 tokens] → ...
```

**代表**：
- **GPT-4o** (OpenAI)：最成功的统一模型，理解+生成文本/图像/音频
- **Chameleon** (Meta)：早期融合，文本和图像交错自回归
- **Emu3** (BAAI)：纯自回归生成图像、文本、视频

**优势**：架构优雅、可扩展、涌现跨模态能力
**挑战**：图像质量仍在追赶专用扩散模型

### 路线 2：混合 AR + Diffusion (Transfusion)

```
核心：文本用自回归，图像用扩散
       共享一个 Transformer 骨干

一次前向传播：
  文本 token → 自回归 loss (next token prediction)
  图像 token → 扩散 loss (denoising / flow matching)
```

**代表**：
- **Transfusion** (Meta, 2024)：[arXiv:2408.11039](https://arxiv.org/abs/2408.11039)
- **Show-o**：自回归文本 + 离散扩散图像

**优势**：结合两种范式的优点，图像质量更高
**挑战**：训练更复杂

### 路线 3：解耦理解+生成 (Janus)

```
核心：理解和生成用不同的视觉编码器
       共享一个 LLM 骨干

理解路径: 图像 → SigLIP 编码器 → LLM
生成路径: LLM → VQ token → VQ 解码器 → 图像

关键洞察：
  理解和生成对视觉表示的需求不同
  → 理解需要语义级表示 (SigLIP)
  → 生成需要像素级表示 (VQ)
  → 强行共享会互相损害
```

**代表**：
- **Janus / Janus-Pro** (DeepSeek, 2024-2025)

**优势**：各路径独立优化
**挑战**：不完全统一

### 路线 4：LLM + 外部扩散模型

```
核心：LLM 做编排和条件生成
       外部扩散模型做图像生成

LLM: 理解用户意图 → 生成详细 prompt → 调用扩散模型
扩散模型: 根据 LLM 生成的 prompt 生成图像
```

**代表**：DALL-E 3 (GPT-4 写 prompt → 扩散模型生图)

**优势**：各组件都是 SOTA
**挑战**：非真正统一，无法无缝交错生成

## 三、视觉 Tokenizer

### 重要性
- 统一 AR 方案的图像质量**直接取决于** VQ Tokenizer 的重建质量
- 这是当前的瓶颈之一

### 主要方案
| Tokenizer | 类型 | 压缩率 | 重建质量 |
|-----------|------|--------|---------|
| VQ-VAE | 离散 | 中 | 中 |
| VQ-GAN | 离散 + 对抗训练 | 中 | 较好 |
| SBER-MoVQGAN | 离散 | 高 | 好 |
| 连续 VAE + Diffusion | 连续 | 灵活 | 最好 |

### 研究方向
- 提高 VQ Tokenizer 的重建质量（关键瓶颈）
- 更高压缩率 → 更少 token → 更快生成
- 视频 Tokenizer (3D)：时间 + 空间同时压缩

## 四、训练方法

### 多任务预训练
```
数据类型：
  1. 交错文本-图像数据 (网页、文档)  → 学习跨模态理解
  2. 纯文本数据                     → 保持语言能力
  3. 纯图像/视频数据                → 学习视觉生成
  4. 指令微调数据                   → 学习任务执行

训练策略：
  方案 A: 从头多模态训练 (Chameleon, Emu3)
  方案 B: 先训练文本 → 再加入视觉 (大多数 VLM)
  方案 C: 联合训练 + 恒定比例混合 (Kimi K2.5)
```

### Kimi K2.5 的联合训练 ★
- 在整个训练过程中以恒定比例混合文本和视觉 token
- **关键发现**：早期融合 + 低比例视觉 token 效果更好
- 与传统 "先文本后视觉" 方案不同
- 结果：视觉和语言能力**同时提升**，无冲突

## 七、Transfusion 深入分析

> 论文：[Transfusion: Predict the Next Token and Diffuse Images with One Multi-Modal Model](https://arxiv.org/abs/2408.11039) (Meta, Waymo, USC, 2024.08)

### 7.1 核心思想：一个 Transformer 同时做 AR 和扩散

Transfusion 的核心洞察是：**文本和图像天然适合不同的生成范式**——文本是离散的、序列化的，适合自回归 (next token prediction)；图像是连续的、空间结构化的，适合扩散 (denoising)。与其强行把图像离散化后用 AR 生成（如 Chameleon），不如让一个 Transformer 同时承载两种目标函数。

```
传统统一 AR 方案（如 Chameleon）:
  图像 → VQ 离散化 → 离散 token → 自回归生成
  问题：VQ 离散化导致信息损失，图像质量受限

Transfusion 方案:
  图像 → VAE 编码 → 连续 latent patch 向量 → 扩散生成
  文本 → 标准 BPE tokenizer → 离散 token → 自回归生成
  两者共享同一个 Transformer 骨干网络
```

### 7.2 训练损失设计

Transfusion 的训练损失由两部分简单相加：

```
L_total = L_LM(文本) + λ · L_DDPM(图像)

其中:
  L_LM   = 标准交叉熵损失 (next token prediction)
           对序列中每个文本 token 计算

  L_DDPM = 扩散去噪损失 (L2 误差)
           对序列中每个图像 latent patch 计算
           具体：给 patch 加随机噪声 → Transformer 预测去噪后的版本 → 计算 L2 距离

  λ      = 平衡系数，控制两个损失的相对权重
```

**关键细节**：
- 文本 token 使用**因果注意力** (causal attention)，只能看到前面的 token
- 图像 patch 在标准因果注意力之上，额外启用**双向注意力** (bidirectional attention)
- 即：同一张图像内的所有 patch 可以互相看到，但只能看到序列中更早出现的文本和其他图像
- 这种"因果 + 图像内双向"的注意力机制，让图像 patch 能够进行充分的空间信息交换

### 7.3 训练数据：文本和图像如何混合

```
训练序列格式：

[文本 token₁] [文本 token₂] ... [BOI] [patch₁] [patch₂] ... [patch_N] [EOI] [文本 token_k] ...

BOI = Begin of Image 标记（特殊离散 token）
EOI = End of Image 标记（特殊离散 token）
patch₁...patch_N = 连续向量（不是离散 token！）
```

**图像编码过程**：
1. 原始图像 → 预训练 VAE 编码器 → latent 空间表示
2. latent 图像按 8x8 网格划分为 patch
3. 每个 patch 压缩为 8 维连续向量
4. 按从左到右、从上到下的顺序排列成序列
5. 前后加上 BOI / EOI 标记嵌入到文本序列中

**数据配比**：
- 7B 参数模型在 2T token 上训练
- 约 1T 文本 token + ~6.92 亿张图像及其 caption（约等于 1T 图像 patch/token）
- 文本与图像约 50:50 的 token 比例

### 7.4 U-Net 增强与推理流程

Transfusion 在 Transformer 基础上引入了 **U-Net 的上/下采样模块** (约 0.27B 额外参数)：
- 编码阶段：U-Net down blocks 将图像 patch 压缩到更少数量（可低至 16 个 patch 表示一张图像）
- 解码阶段：U-Net up blocks 将生成的 patch 还原到完整分辨率
- U-Net 内部自带双向注意力，进一步增强图像 patch 间的信息流

**推理（图像生成）流程**：
```
1. 模型自回归生成文本 token，直到产生 BOI token
2. 进入扩散解码模式：
   a. 初始化一组随机噪声 patch 向量
   b. 反复将序列（文本上下文 + 当前噪声 patch）送入 Transformer
   c. Transformer 预测去噪后的 patch → 更新 patch → 重复
   d. 经过 T 步去噪，得到干净的 latent patch 向量
3. U-Net up blocks 将 patch 映射回 latent 图像
4. VAE 解码器将 latent 图像解码为最终 RGB 像素图像
5. 遇到 EOI token，切换回自回归文本模式
```

### 7.5 Scaling Law 与关键实验结论

| 发现 | 详情 |
|------|------|
| **显著优于离散化方案** | 在相同计算量下，Transfusion 的图像生成质量远超将图像 VQ 离散化后用 AR 生成的方案 |
| **计算效率极高** | 文生图任务使用仅约 1/34 的计算量即可达到可比 FID 分数；文本任务使用约 50-60% 的 FLOPs 即可匹配同级别模型 |
| **模型规模可扩展** | 从小模型到 7B，文本和图像的 loss 均呈现良好的 scaling law |
| **16 patch 极致压缩** | 引入 U-Net 编/解码层后，每张图像可压缩到仅 16 个 latent patch，大幅降低序列长度 |

## 八、Janus / Janus-Pro 深入分析

> 论文：[Janus: Decoupling Visual Encoding for Unified Multimodal Understanding and Generation](https://arxiv.org/abs/2410.13848) (DeepSeek, CVPR 2025)
> 论文：[Janus-Pro: Unified Multimodal Understanding and Generation with Data and Model Scaling](https://arxiv.org/abs/2501.17811) (DeepSeek, 2025.01)

### 8.1 核心问题：为什么理解和生成需要不同的视觉表示？

这是 Janus 系列最重要的洞察。先前的统一模型（如 Chameleon, Emu3）使用**同一个视觉编码器**同时服务于理解和生成，但这两个任务对视觉信息的需求截然不同：

```
视觉理解任务的需求：
  "这张图片里有什么？" → 需要高层语义信息
  → 物体类别、空间关系、场景语义
  → 需要丢弃低层纹理细节，提取抽象概念
  → 适合 SigLIP 这类对比学习编码器（语义丰富、空间信息有损）

视觉生成任务的需求：
  "画一只猫" → 需要低层像素级信息
  → 精确的纹理、颜色、空间布局
  → 需要保留细粒度的视觉细节
  → 适合 VQ Tokenizer（空间信息保留、语义相对有限）

冲突所在：
  如果共享同一个视觉编码器——
  → 若偏向语义（如 SigLIP），生成任务丢失细节
  → 若偏向像素（如 VQ），理解任务丢失语义
  → 两者无法兼得，性能互相拖累
```

### 8.2 解耦架构设计

Janus 的解决方案是**解耦视觉编码**，使用两个独立的视觉编码器，但共享一个 LLM 骨干：

```
                    ┌─────────────────────────┐
                    │   共享 LLM 骨干 (DeepSeek)  │
                    │   统一的语言-视觉推理     │
                    └──────┬──────────┬────────┘
                           │          │
              ┌────────────┴──┐  ┌────┴────────────┐
              │  理解编码器     │  │  生成编码器/解码器 │
              │  SigLIP-Large  │  │  VQ Tokenizer    │
              │  Patch16/384   │  │  (离散码本)       │
              └───────┬───────┘  └────┬─────────────┘
                      │               │
               输入图像(理解)     输出图像(生成)
```

**SigLIP 编码器（理解路径）**：
- 使用 SigLIP-Large-Patch16/384 预训练模型
- 输入分辨率 384x384，输出高维语义特征
- 通过 MLP adaptor 投射到 LLM 的嵌入空间
- 优点：经过大规模图文对比学习，语义表征极其丰富

**VQ Tokenizer（生成路径）**：
- 将图像编码为离散 token ID 序列
- LLM 自回归预测这些 token ID
- VQ 解码器将 token ID 序列还原为图像
- 优点：保留空间细节，适合像素级重建

### 8.3 Janus-Pro 的改进（相对于 Janus）

Janus-Pro 在三个维度上进行了优化：

**1. 训练策略优化**：
```
三阶段训练流程：

Stage I: 适配器训练 + 图像头训练
  → 训练时长增加，在 ImageNet 上充分训练
  → 使模型建立基础的视觉-语言对齐

Stage II: 统一预训练
  → 关键改动：放弃 ImageNet 数据，直接使用密集描述的文生图数据
  → 更高效地学习基于描述生成图像的能力

Stage III: 监督微调 (SFT)
  → 多模态理解 : 文本 : 文生图 的数据比例从 7:3:10 调整为 5:1:4
  → 更均衡的任务分配
```

**2. 数据规模扩展**：
- 多模态理解数据增加约 9000 万样本
- 视觉生成数据增加约 7200 万合成美学样本
- 统一预训练阶段真实数据:合成数据 = 1:1

**3. 模型规模扩展**：
- Janus-Pro-1B 和 Janus-Pro-7B 两个尺寸
- 7B 版本在 MMBench 达到 79.2（超越同类统一模型）
- GenEval 文生图评测 0.80（超越 DALL-E 3 的 0.67 和 SD3 Medium 的 0.74）

## 九、视觉 Tokenizer 深入解析

> 视觉 Tokenizer 是统一 AR 模型的核心瓶颈。其重建质量直接决定生成图像的上限。

### 9.1 VQ-VAE：向量量化的工作原理

**VQ-VAE** (Vector Quantized Variational AutoEncoder) 由 DeepMind 于 2017 年提出，是将连续图像转化为离散 token 的基础方法。

```
VQ-VAE 的三个核心组件：

1. 编码器 (Encoder)
   输入图像 x → 连续特征图 z_e(x) ∈ R^{H'×W'×D}
   通过卷积网络将图像压缩到低维连续表示

2. 码本 (Codebook)
   一组可学习的嵌入向量 E = {e₁, e₂, ..., e_K}
   K = 码本大小 (常见: 1024, 8192, 16384)
   每个 e_i ∈ R^D

3. 解码器 (Decoder)
   输入量化后的特征图 z_q(x) → 重建图像 x̂
```

**向量量化过程**：
```
对编码器输出 z_e(x) 的每个空间位置的向量 z:

1. 计算 z 到码本中每个嵌入向量的距离:
   d(z, e_k) = ||z - e_k||²    对所有 k ∈ {1,...,K}

2. 找到最近的码本向量:
   k* = argmin_k ||z - e_k||²

3. 用最近的码本向量替换 z:
   z_q = e_{k*}

这就是 "量化"：将连续空间中的任意向量，映射到码本中最近的离散向量
```

**VQ-VAE 的损失函数**：
```
L_total = L_recon + L_codebook + β · L_commit

三个组成部分：

1. 重建损失 (Reconstruction Loss):
   L_recon = ||x - Decoder(z_q)||²
   → 让解码器能从量化表示重建原图

2. 码本损失 (Codebook Loss):
   L_codebook = ||sg[z_e(x)] - e||²
   → 将码本向量 e 推向编码器输出 z_e(x)
   → sg[] 是 stop-gradient 算子，不对编码器求梯度
   → 只更新码本嵌入

3. 承诺损失 (Commitment Loss):
   L_commit = ||z_e(x) - sg[e]||²
   → 反向约束：让编码器输出 z_e(x) 靠近所选的码本向量
   → sg[] 停止对码本求梯度，只更新编码器
   → β 通常取 0.25，控制承诺损失的权重

为什么需要承诺损失？
  因为码本的嵌入空间是无量纲的，如果编码器训练得比码本快，
  编码器输出可能无限远离码本向量，导致量化完全失效。
  承诺损失强制编码器 "承诺" 留在码本向量附近。
```

**梯度传递问题与 Straight-Through Estimator**：
```
核心难题：argmin 操作不可微！

z_q = e_{argmin_k ||z - e_k||²}  ← 这一步没有梯度

解决方案：Straight-Through Estimator (STE)
  前向传播：使用量化后的 z_q
  反向传播：直接将 z_q 的梯度复制给 z_e
  即 ∂L/∂z_e ≈ ∂L/∂z_q

这意味着编码器"假装"量化没有发生，直接接收来自解码器的梯度
```

**码本崩塌问题 (Codebook Collapse)**：
```
问题：训练过程中大量码本向量从未被选中（"死码"）
     极端情况下只有少量码本向量被使用 → 有效码本大小远小于 K

原因：
  - 编码器输出集中在嵌入空间的某个区域
  - 远离该区域的码本向量永远不会被最近邻选中
  - 得不到梯度更新 → 永远不会被使用

解决方案：
  1. EMA 更新 (Exponential Moving Average)
     → 用指数移动平均替代梯度下降来更新码本
     → e_k ← γ · e_k + (1-γ) · mean(被分配到 e_k 的所有 z)
     → 更稳定的码本更新

  2. 死码重初始化
     → 监控每个码本向量的使用频率
     → 长期未使用的向量重新初始化为当前 batch 中的随机编码器输出

  3. K-means 初始化
     → 训练开始前用 K-means 聚类初始化码本
     → 保证初始码本就覆盖编码器输出的分布
```

### 9.2 VQ-GAN：加入判别器提升重建质量

**VQ-GAN** (Taming Transformers, Esser et al., 2021) 在 VQ-VAE 基础上引入了两个关键改进：

```
VQ-GAN = VQ-VAE + 感知损失 + 对抗训练

改进 1: 感知损失 (Perceptual Loss)
  不再仅用像素级 L2 误差衡量重建质量
  → 用预训练 VGG 网络提取原图和重建图的高层特征
  → 计算特征空间的 L2 距离
  → 让重建图在 "感知" 层面更接近原图（而非像素精确匹配）

  L_perceptual = ||VGG(x) - VGG(x̂)||²

改进 2: Patch-based 判别器 (Discriminator)
  引入 PatchGAN 判别器：
  → 将图像划分为多个 patch
  → 判别器对每个 patch 判断真假
  → 迫使重建图像在局部纹理上更加逼真

  L_GAN = log D(x) + log(1 - D(x̂))
```

**VQ-GAN 的完整损失**：
```
L_VQ-GAN = L_recon + L_perceptual + L_codebook + β·L_commit + λ_GAN·L_GAN

对比 VQ-VAE：
  VQ-VAE:  像素级 L2 重建 → 模糊、缺乏细节
  VQ-GAN:  感知损失 + 对抗损失 → 纹理清晰、细节丰富
```

**VQ-GAN 的两阶段使用**：
```
阶段 1: 训练 VQ-GAN 本身
  → 编码器 + 码本 + 解码器 + 判别器联合训练
  → 目标：学到高质量的图像离散化和重建

阶段 2: 训练自回归 Transformer
  → 冻结 VQ-GAN，用编码器把图像转成 token 序列
  → 用 Transformer 学习 token 序列的分布 (next token prediction)
  → 推理时：Transformer 生成 token 序列 → VQ-GAN 解码器重建图像
```

### 9.3 码本大小与压缩率对生成质量的影响

```
码本大小 K 的影响：

K 太小 (如 256-512):
  → 每个 token 只能表示少量视觉模式
  → 重建误差大，细节丢失严重
  → 但自回归建模更容易（词表小，分布简单）

K 太大 (如 65536-131072):
  → 每个 token 可以精细表示视觉细节
  → 重建质量高
  → 但自回归建模更难（词表大，分布复杂）
  → 需要更多计算量才能收敛
  → 更容易出现码本崩塌

实验发现（2024-2025 研究）:
  - 计算量有限时：小码本 (1K) 反而比大码本 (131K) 好 5-10 gFID
  - 计算量充足时：大码本 (131K) 最终超越小码本
  - 256 token/image 是计算最优的分辨率，直到触及 ~2.5 gFID 瓶颈

压缩率的影响：

低压缩率 (如 256×256 → 32×32 = 1024 tokens):
  → 保留更多空间信息
  → 图像质量更高
  → 但序列太长，自回归生成速度慢

高压缩率 (如 256×256 → 8×8 = 64 tokens 甚至 32 tokens):
  → 序列短，生成快
  → 但信息损失大
  → TiTok (2024) 证明最少可到 32 个 token，运行速度提升 10 倍
  → 但需要更强大的解码器来补偿压缩损失
```

### 9.4 当前瓶颈与研究方向

```
瓶颈 1: 离散 token 的信息容量限制
  → 每个离散 token 只能承载 log₂(K) bit 信息
  → 对于 K=8192，每 token 仅 13 bits
  → 远不足以表达图像的全部细节

  → 研究方向: 多码本策略 (Multi-Codebook)
    UniTok (2025): 使用多个码本并行量化
    → 每个位置用多个码本向量组合表示
    → 有效信息容量 = N × log₂(K)（N 为码本数）
    → ImageNet rFID 0.38，零样本准确率 78.6%

瓶颈 2: 重建质量 vs 生成质量的 trade-off
  → 令人意外的发现：更好的重建不一定带来更好的生成
  → 原因：过于精细的 tokenizer 可能让 AR 模型更难学习分布

  → 研究方向: 压缩-生成权衡导航
    论文 "When Worse is Better" (2024) 系统研究了这个 trade-off

瓶颈 3: 高压缩率下的解码器能力
  → 高压缩率 tokenizer 需要极其强大的解码器
  → 解码器本身可能需要是一个生成模型

  → 研究方向:
    - Bit-level 生成 (BAR, 2025): 不预设码本大小，逐 bit 生成 token
      → ImageNet-256 达到 0.99 gFID (SOTA)
    - 1D Tokenizer (TiTok): 打破 2D 空间结构，用 1D 序列极致压缩
    - 统一 Tokenizer (UniTok): 同时优化理解和生成目标
```

## 十、GPT-4o 的图像生成方法

> OpenAI 于 2025 年 3 月推出 GPT-4o 原生图像生成能力，这是统一多模态模型最成功的商业实现。

### 10.1 与之前方案的本质区别

```
DALL-E 3 时代（管道方案）:
  用户输入 → GPT-4 理解意图 → 改写 prompt → 调用外部扩散模型 → 返回图像
  问题：两个独立模型，无法真正交错文本和图像

GPT-4o 原生图像生成:
  用户输入 → 同一个 GPT-4o → 自然切换文本/图像输出 → 统一生成
  优势：模型内部共享知识，文本理解能力直接指导图像生成
```

### 10.2 推测的技术架构（基于 Transfusion 框架）

虽然 OpenAI 未公开完整技术细节，但根据公开信息和研究者分析，GPT-4o 极可能基于 Transfusion 式架构：

```
GPT-4o 图像生成的推测流程：

1. 自回归文本生成阶段
   → 模型正常生成文本 token
   → 当模型决定需要输出图像时，生成特殊的 BOI (Begin of Image) token

2. 扩散图像生成阶段
   → BOI 触发后，模型切换到扩散解码模式
   → 图像以 latent patch（连续向量）的形式表示，而非离散 token
   → 通过多步去噪迭代逐步从噪声中生成清晰图像
   → 每一步去噪都通过 Transformer 的完整前向传播
   → 文本上下文作为条件，全程引导图像生成

3. 图像解码阶段
   → 最终的 latent patch 向量经 VAE 解码器转换为像素图像
   → 遇到 EOI (End of Image) token，切换回文本模式

关键优势：
  → 文本和图像共享同一个 Transformer 的全部知识
  → 模型对 "猫" 的语义理解直接影响画猫的方式
  → 支持多轮对话中逐步修改图像（因为有上下文记忆）
  → 支持图像中精确渲染文本（因为语言模型天然理解文字）
```

### 10.3 GPT-4o 图像生成的独特能力

```
传统扩散模型难以做到的：

1. 图像内文字渲染
   → 扩散模型对文字渲染一直很弱
   → GPT-4o 因为本质是语言模型，天然理解字符序列
   → 可以在图像中准确渲染文字、标签、表格

2. 多轮一致性编辑
   → "给这张图加个帽子" → "再把背景换成海滩"
   → 扩散模型需要额外的 inpainting/editing 模块
   → GPT-4o 在对话上下文中自然保持图像一致性

3. 跨模态推理生成
   → "画一张图解释相对论" → 需要先理解物理概念，再视觉化表达
   → 统一模型天然将知识从文本域迁移到视觉域

4. 流式输出
   → AR 生成可以逐步输出部分结果
   → 用户可以在生成过程中看到图像逐步清晰化
   → 比扩散模型的全量输出更适合交互式应用
```

### 10.4 当前局限与挑战

```
1. 过度安全限制：许多合理的图像请求被拒绝
2. 风格多样性：生成风格相对固定，不如专用扩散模型灵活
3. 分辨率限制：原生分辨率可能受限于 latent patch 的数量
4. 生成速度：扩散去噪的多步迭代导致图像生成较慢
5. 幻觉问题：可能生成不符合物理规律的图像细节
```

## 十一、Kimi K2.5 联合训练洞察

> 论文：[Kimi K2.5: Visual Agentic Intelligence](https://arxiv.org/abs/2602.02276) (Moonshot AI, 2025.02)

### 11.1 核心发现：早期融合 + 恒定低比例优于后期大量灌入

传统多模态训练的主流做法是"先文本后视觉"：
```
传统方案（阶段式训练）:
  Phase 1: 纯文本预训练 → 100% 文本 token → 建立语言能力
  Phase 2: 加入视觉数据 → 50-70% 视觉 token → 训练多模态能力

  问题：
  → 视觉数据集中在训练后期
  → 可能破坏已学到的语言能力（灾难性遗忘）
  → 模态融合不够充分
```

Kimi K2.5 的突破性发现：
```
Kimi K2.5 方案（恒定比例联合训练）:
  全程: 90% 文本 token + 10% 视觉 token
  → 从训练一开始就混入视觉数据
  → 视觉占比低但持续存在
  → 总量约 15 万亿 (15T) 混合 token

消融实验关键结论：
  固定总预算下（文本+视觉 token 总量不变）：
  → 视觉比例的具体数值对最终多模态性能影响不大
  → 但注入时机影响很大！
  → 早期融合 + 低视觉比例 >> 后期融合 + 高视觉比例

  例如：
  → 从头 10:90 (视觉:文本) 全程训练  ✓ 最优
  → 先纯文本训练 50%，再 50:50 训练  ✗ 较差
```

### 11.2 为什么早期融合更好？

```
假说 1: 跨模态表征对齐
  → 从训练初期就让文本和视觉共享表征空间
  → 模型的底层特征从一开始就是"多模态原生"的
  → 后期注入视觉数据需要改造已固化的纯文本表征，代价更高

假说 2: 课程学习效应
  → 低比例视觉数据相当于"轻量级"的持续多模态信号
  → 模型不会被视觉数据"冲击"导致语言能力退化
  → 类似于人类从婴儿期就同时接收视觉和语言信号

假说 3: 梯度平衡
  → 低比例视觉 token 意味着视觉相关梯度信号较弱
  → 不会显著干扰文本任务的优化
  → 但持续提供跨模态学习信号
```

### 11.3 视觉 RL 反向提升文本能力

Kimi K2.5 的另一个反直觉发现：**视觉强化学习 (Visual RL) 能提升纯文本任务性能**。

```
实验数据（视觉 RL 前 → 后）:
  MMLU-Pro (纯文本知识):  84.7% → 86.4%  ↑ 1.7%
  GPQA-Diamond (推理):    84.3% → 86.4%  ↑ 2.1%
  LongBench v2 (长文本):  56.7% → 58.9%  ↑ 2.2%
```

**原因分析**：
```
Kimi K2.5 采用的联合多模态 RL 框架：
  → 不按输入模态划分 RL 专家（传统做法）
  → 而是按能力划分：知识、推理、编码、Agent 等
  → 每个领域专家同时从纯文本和多模态查询中学习
  → 奖励模型 (GRM) 也跨模态优化

效果：
  → 视觉推理任务中学到的 "如何一步步分析复杂问题"
  → 自然迁移到文本推理任务
  → 跨模态能力迁移是双向的
  → 这打破了 "多模态训练会损害语言能力" 的固有认知
```

### 11.4 三阶段预训练流程

```
Stage 1: ViT 独立训练
  → 训练原生分辨率的视觉编码器 (Vision Transformer)
  → 建立强大的视觉特征提取基础

Stage 2: 联合预训练
  → 基于 Kimi K2 语言模型 checkpoint
  → 持续混合文本和视觉 token，恒定比例
  → 同时提升语言和多模态能力

Stage 3: 中期训练
  → 高质量数据精炼
  → 长上下文激活
  → 能力微调和对齐
```

## 十二、Chameleon 补充说明

> 论文：[Chameleon: Mixed-Modal Early-Fusion Foundation Models](https://arxiv.org/abs/2405.09818) (Meta FAIR, 2024)

### 12.1 纯统一 AR 的代表

Chameleon 是**最彻底的早期融合统一 AR 模型**之一，它的设计哲学与 Transfusion/Janus 形成鲜明对比：

```
Chameleon 的设计理念：
  → 一切都是 token
  → 图像用 VQ 离散化为 token（与文本 token 地位完全平等）
  → 一个 Transformer，一种目标函数 (next token prediction)
  → 不需要扩散，不需要解耦，极致简洁

训练规模：
  → 7B 和 34B 两个尺寸
  → 4.4 万亿 token（文本 + 图文对 + 交错图文序列）
  → 500 万 GPU 小时

训练稳定性挑战：
  → 纯早期融合训练极不稳定（loss spike 频繁）
  → Chameleon 引入了多项训练技巧:
    · QK-Norm: 对注意力的 Q/K 向量做归一化
    · 重新排列 LayerNorm 的位置
    · Dropout 调整
  → 这些技巧是使大规模早期融合训练可行的关键
```

### 12.2 Chameleon 的局限与启示

```
局限：
  → 图像生成质量受限于 VQ Tokenizer 的重建质量
  → 与 Transfusion 对比：在相同计算量下图像质量明显更差
  → 证明了单纯用 AR + 离散 token 生成图像的天花板

启示：
  → 证明了早期融合是可行的，但需要精心的训练工程
  → 引出了 Transfusion 的核心动机：不一定要把图像强行离散化
  → 引出了 Janus 的核心动机：不一定要共享视觉编码器
```
