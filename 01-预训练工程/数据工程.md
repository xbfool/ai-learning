# 数据工程

> 数据质量决定模型质量上限。本文梳理从原始网页到高质量预训练 Token 的完整流水线。

## 一、数据处理流水线

```
原始网页 (Common Crawl)
  │
  ├── 1. URL 过滤        → 移除低质量域名
  ├── 2. 文本提取        → HTML → 纯文本 (trafilatura, resiliparse)
  ├── 3. 语言识别        → fastText 语言分类器
  ├── 4. 去重 (Dedup)    → MinHash / SimHash / Exact Dedup
  ├── 5. 质量过滤        → 基于模型的分类器 (perplexity, 教育性)
  ├── 6. PII 清洗        → 移除个人身份信息
  ├── 7. 安全过滤        → 移除有毒/违规内容
  └── 8. 数据混合        → 按比例混合不同数据源
      │
      → 高质量预训练 Token (5T-30T)
```

## 二、核心开源数据集

### FineWeb (HuggingFace, 2024)
- **规模**：15T tokens
- **论文**：[arXiv:2406.17557](https://arxiv.org/abs/2406.17557)
- **核心创新**：
  - 教育内容分类器 (educational quality classifier) 过滤高质量数据
  - 基于 MinHash 的大规模去重
  - FineWeb-Edu 子集：专注高教育价值内容
- **关键发现**：数据质量 > 数据数量；经过良好过滤的 1.3T tokens 可以匹配未过滤的 15T tokens

### DCLM (DataComp for Language Models, 2024)
- **论文**：[arXiv:2406.11794](https://arxiv.org/abs/2406.11794)
- **核心创新**：
  - 开放数据筛选竞赛/基准
  - 基于 fastText 的质量分类器，用高质量参考数据（如 Wikipedia、教科书）训练
  - 证明 **model-based filtering** 是最有效的数据筛选策略
- **关键发现**：用小型分类器（fastText）做数据筛选，性价比远高于用大模型

### Dolma 3 (AI2, 2025)
- 用于 OLMo 3 训练
- 起始规模：~9.3T tokens 的网络抓取
- 最终混合：~5.9T tokens，重点加码代码和数学
- **完全公开**：数据处理代码 + 原始数据全部开源

## 三、合成数据 (Synthetic Data)

### 动机
- 高质量自然数据越来越稀缺
- 合成数据可以针对性补充特定领域（数学推理、代码、指令遵循）

### 方法
1. **Rephrase/Reformulate**：用 LLM 改写已有数据（Kimi K2 采用此策略）
2. **Self-Instruct**：LLM 自动生成指令-响应对
3. **Evol-Instruct**：逐步提高指令复杂度（WizardLM）
4. **数学/代码验证**：生成问题 + 用规则验证答案正确性
5. **蒸馏**：从大模型生成数据训练小模型（DeepSeek-R1 蒸馏到 1.5B-70B）

### Kimi K2 的合成数据策略
- 设计了精细的 **rephrasing pipeline**
- 在不引入显著过拟合的情况下增加高质量 token 数量
- 显著提升了 token 利用效率

## 四、数据混合 (Data Mixing)

### 典型混合比例
| 数据类型 | 占比 | 来源 |
|---------|------|------|
| 网页文本 | 50-70% | Common Crawl 过滤后 |
| 代码 | 10-20% | GitHub, Stack |
| 数学/科学 | 5-10% | arXiv, 教科书, 合成 |
| 书籍 | 5-10% | Project Gutenberg 等 |
| 百科 | 3-5% | Wikipedia |
| 对话/论坛 | 2-5% | Reddit, StackOverflow |

### 多阶段训练中的数据混合
- **阶段 1 (主训练)**：通用混合，大规模训练
- **阶段 2 (中训练/退火)**：提高高质量数据占比（代码、数学、推理）
- **阶段 3 (长上下文)**：长文档数据占主导
- OLMo 3 明确采用三阶段策略：主训练 → 数学/代码强化 → 长上下文扩展

## 五、关键经验

1. **去重极其重要**：重复数据会导致 memorization，降低泛化能力
2. **质量过滤决定上限**：一个好的数据分类器价值 > 10x 更多的数据
3. **数据混合是门艺术**：需要大量消融实验确定最优比例
4. **合成数据是双刃剑**：能补充稀缺领域，但过多会导致 model collapse
5. **数据溯源和版权**：AI2 坚持完全开放数据链条，是行业标杆
