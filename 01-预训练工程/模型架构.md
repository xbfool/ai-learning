# 模型架构

> 架构创新是提升模型效率和能力的核心杠杆。本文梳理 2024-2026 年的关键架构趋势。

## 一、架构全景

```
模型架构
├── Dense Transformer    → Llama 3, OLMo 3 (经典但计算成本高)
├── MoE (混合专家)       → DeepSeek-V3, Kimi K2.5, GLM-5, Step 3.5 Flash
├── 注意力优化           → MLA, GQA, MQA (KV Cache 压缩)
├── 替代架构             → Mamba/SSM, RWKV, RetNet (线性复杂度)
├── 多模态架构           → DiT (图像/视频), Transfusion (统一)
└── 长上下文             → RoPE 扩展, Ring Attention, 渐进训练
```

## 二、MoE (Mixture of Experts) 深入解析

### 核心思想
- 每层有 N 个 "专家"（每个专家是一个 FFN）
- 每个 token 只激活 Top-K 个专家
- 效果：**总参数大，激活参数少** → 用更少计算获得更多容量

### 关键模型对比

| 模型 | 总参数 | 激活参数 | 专家数 | Top-K | 特殊设计 |
|------|--------|----------|--------|-------|---------|
| Mixtral 8x7B | 46.7B | 12.9B | 8 | 2 | 经典设计 |
| DeepSeek-V3 | 671B | 37B | 256 routed + 1 shared | 8 | 无辅助损失负载均衡 |
| Kimi K2.5 | 1T | 32B | 384 routed | - | 超稀疏 MoE |
| GLM-5 | 744B | 44B | MoE | - | 稀疏注意力 |
| Step 3.5 Flash | 197B | 11B | 288 routed + 1 shared | - | SWA 3:1 |

### 核心技术细节

#### 负载均衡 (Load Balancing)
- **传统方法 (Auxiliary Loss)**：添加辅助损失项惩罚不均匀的专家利用率
  - 问题：辅助损失与主训练目标冲突，损害模型质量
- **DeepSeek-V3 创新 (Auxiliary-Loss-Free)**：
  - 在路由门控中引入动态偏置项 (bias term)
  - 偏置项根据每个专家的利用率自动调整
  - 无需辅助损失，避免质量-均衡的权衡
  - **这是一个重要突破**，被后续模型广泛采用

#### 细粒度专家 (Fine-Grained Experts)
- DeepSeek 系列使用更多、更小的专家（256 个 vs 传统 8 个）
- 更灵活的专家组合，更精细的知识分工
- 每个 token 激活 8 个专家，组合空间为 C(256,8) ≈ 4.4 × 10^12

#### 共享专家 (Shared Experts)
- 部分专家始终被激活（不经过门控）
- 捕获所有输入共享的通用知识
- 剩余路由专家可以专注于特化知识
- 稳定训练，减少路由专家之间的信息冗余

### 专家并行 (Expert Parallelism, EP)
```
设备 A: 专家 0-63     设备 B: 专家 64-127
设备 C: 专家 128-191  设备 D: 专家 192-255

Token 路由：
1. All-to-All 通信：token 发送到持有目标专家的设备
2. 专家计算：各设备处理分配到的 token
3. All-to-All 通信：结果发回源设备
```

## 三、注意力机制优化

### Multi-Head Latent Attention (MLA) - DeepSeek-V2/V3
- **问题**：标准 Multi-Head Attention 的 KV Cache 占用大量 GPU 显存
- **方案**：将 Key 和 Value 投影到低秩潜空间
  - KV Cache 从 O(n × d_model) 降到 O(n × d_latent)，d_latent << d_model
- **效果**：KV Cache 减少 93.3%（从 d_model=7168 到 d_latent=512）

### Grouped Query Attention (GQA)
- 多个 Query Head 共享一组 Key-Value Head
- Llama 2/3, Mistral, Gemma 使用
- 例：32 个 Query Head, 8 个 KV Head → KV Cache 减少 4x

### Multi-Query Attention (MQA)
- 所有 Query Head 共享同一组 KV Head → 最激进的压缩

### 对比
| 方法 | KV Head 数 | 压缩比 | 质量影响 |
|------|-----------|--------|---------|
| MHA | = Query Heads | 1x | 基线 |
| GQA | Query/N | 4-8x | 轻微 |
| MQA | 1 | 32x+ | 中等 |
| MLA | 潜空间 | 14x | 几乎无 |

## 四、多 Token 预测 (Multi-Token Prediction)
- **DeepSeek-V3** 训练时预测多个未来 token
- 增加训练信号密度
- 副产品：MTP 头可直接用于推测解码 (Speculative Decoding)
- Meta 也在 Llama 研究中探索此方向

## 五、替代架构

### Mamba / State Space Models (SSM)
- **核心**：用状态空间模型替代注意力，实现线性复杂度
- Mamba 2 进一步优化
- **优势**：推理速度快（无 KV Cache 问题），长序列高效
- **劣势**：在复杂推理任务上仍不如 Transformer
- **混合方案**：Jamba (AI21) = Mamba + Attention 交替层

### RWKV (Eagle/Finch)
- 融合 RNN 和 Transformer 的优点
- 线性推理复杂度
- 社区活跃，持续迭代

## 六、长上下文训练

### RoPE 扩展
- **YaRN**：高效的 RoPE 缩放方法
- **LongRoPE**：扩展到 2M tokens（Microsoft）
- 关键：修改位置编码的频率基数/缩放因子

### 渐进式上下文扩展
- Llama 3.1 的方案：8K → 128K 分阶段训练
- 先用短上下文训练主体，再用长文档微调

### Ring Attention
- 将长序列切分到多个设备
- 设备之间环形传递 KV blocks
- 实现跨设备的完整注意力计算
- 用于 Context Parallelism (CP)

## 七、架构选择建议

| 场景 | 推荐架构 |
|------|---------|
| 万亿参数级模型 | MoE + MLA + 细粒度专家 |
| 中小规模 (7B-70B) | Dense Transformer + GQA |
| 极端推理效率 | SSM/Mamba 混合 |
| 长上下文 (>128K) | Ring Attention + RoPE 扩展 |
| 多模态 (图/视频) | DiT (Diffusion Transformer) |
