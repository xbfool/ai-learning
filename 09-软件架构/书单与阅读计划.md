# 书单与阅读计划

> 更新：2026-03-16
> 原则：不贪多，分层读。地基打牢比什么都重要。

## 第一层：地基（必读）

### 1. Designing Data-Intensive Applications (DDIA)
- **作者**: Martin Kleppmann
- **为什么第一本读这个**: 分布式系统的底层原理全在这里。复制、分区、一致性模型、事务、流处理。不管你做什么系统，这些是绕不过去的。
- **读法**: 每天 30 分钟，章节式精读。不求快，求理解。
- **预计周期**: 2-3 个月

### 2. Fundamentals of Software Architecture, 2nd Edition (2025)
- **作者**: Mark Richards & Neal Ford
- **定位**: 架构思维的系统入门。第二版新增了 AI、Team Topologies 等章节。
- **覆盖**: 架构特性（可用性、可扩展性、可维护性）、模块化、耦合/内聚、架构风格（分层、微内核、微服务、modular monolith）
- **读法**: 周末集中读，每章对照自己的项目做笔记
- **预计周期**: 1-2 个月

### DDIA + Fundamentals 的搭配读法

```
周一到周五：DDIA 每天一小节（理论为主）
周末：Fundamentals 一章（架构风格、模式为主）

两本交叉读，DDIA 给你底层认知，Fundamentals 给你设计词汇。
```

## 第二层：与当前工作直接相关（项目驱动，边用边读）

### 3. Software Architecture: The Hard Parts (2021)
- **作者**: Neal Ford, Mark Richards, Pramod Sadalage, Zhamak Dehghani
- **定位**: 专讲分布式系统里最难的**权衡决策**
- **核心内容**: 服务拆分粒度、数据所有权、耦合 vs 内聚、分布式事务
- **读法**: 遇到拆分/合并决策时翻对应章节

### 4. Building Agentic AI Systems (2025)
- **作者**: Wrick Talukdar & Anjanava Biswas (Packt)
- **定位**: Agent 控制循环、多步规划、工具集成、coordinator/worker/delegator 模式
- **读法**: 做 Agent 架构设计时参考

### 5. AI Engineering (2025)
- **作者**: Chip Huyen
- **定位**: AI 系统的生产级工程实践。模型版本管理、LLM 编排、RAG 管线。
- **读法**: 做全链路优化时参考

### 6. Building Applications with AI Agents (2025, O'Reilly)
- **定位**: Agent 类型分类、工具集成模式、记忆管理、编排架构
- **与 #4 的区别**: 这本偏应用层设计，#4 偏底层控制循环
- **读法**: 设计新 Agent 功能时查阅

## 第三层：进阶深化（打完地基后选读）

### 7. Designing Distributed Systems, 2nd Edition (2025)
- **作者**: Brendan Burns
- **定位**: 分布式系统设计模式手册。Sidecar、Ambassador、Adapter、Leader Election。
- **适合**: 需要具体模式参考时翻阅

### 8. Building Evolutionary Architectures (2023)
- **作者**: Neal Ford, Rebecca Parsons, Patrick Kua, Pramod Sadalage
- **定位**: 架构如何随业务演进，适应度函数，增量变更。
- **适合**: 思考项目长期演进方向时阅读

### 9. Software Architecture in Practice, 4th Edition (2024)
- **作者**: Len Bass, Paul Clements, Rick Kazman
- **定位**: 学术界公认的架构教材。质量属性场景、架构评估方法 (ATAM)。
- **适合**: 想系统化学习架构评估方法时阅读
