# 语音合成 (Text-to-Speech)

> 2025 年 TTS 技术全面进入人类级质量，从神经编解码器语言模型到 Flow Matching，开源与闭源齐头并进。

---

## 一、技术路线全景

```
TTS 技术路线演进：

第一代：拼接/参数化合成 (2010s)
  WaveNet → Tacotron 2 → 高质量但训练慢

第二代：端到端神经 TTS (2020-2022)
  VITS → 端到端文本到波形
  FastSpeech 2 → 非自回归快速合成

第三代：神经编解码器语言模型 (2023-)  ★★★ 当前主流
  VALL-E → 将 TTS 转化为条件语言建模
  核心思路：文本 → 离散音频 Token → 波形
  代表：VALL-E, Fish Speech/OpenAudio S1, IndexTTS

第四代：Flow Matching + 扩散 (2024-)
  用连续流匹配替代自回归
  代表：F5-TTS, CosyVoice 2, Matcha-TTS

第五代：LLM 原生语音 (2025-)
  大语言模型直接理解和生成语音
  代表：GPT-4o Voice, Gemini Native Audio
```

### 核心技术范式对比

| 范式 | 原理 | 速度 | 质量 | 代表 |
|------|------|------|------|------|
| 神经编解码器 LM | 文本→离散音频码→波形 | 中 | 很高 | VALL-E, Fish Speech |
| Flow Matching | 连续流匹配 + DiT | 快 | 很高 | F5-TTS, CosyVoice 2 |
| 掩码生成 Codec | 非自回归掩码预测 | 快 | 很高 | MaskGCT |
| Dual-AR | 快/慢双自回归 | 中快 | 最高 | OpenAudio S1 |
| LLM 骨干 | LLM + 音频解码器 | 中 | 很高 | CosyVoice 2, GPT-4o-mini-tts |

## 三、开源 TTS 生态 ★★★

### 3.1 CosyVoice 2 (阿里 FunAudioLLM)

> 流式/非流式统一框架，150ms 首包延迟。

| 项目 | 详情 |
|------|------|
| **团队** | 阿里巴巴 FunAudioLLM |
| **论文** | [arXiv:2412.10117](https://arxiv.org/abs/2412.10117) (2024.12) |
| **参数** | 0.5B (Qwen2.5-0.5B 骨干) |
| **架构** | LLM 骨干 + 有限标量量化 (FSQ) + 分块因果流匹配 (chunk-aware causal flow matching) |
| **流式延迟** | 150ms 首包延迟 |
| **效果** | 发音错误率比 CosyVoice 1.0 降低 30-50%；Seed-TTS hard 测试集最低 CER |
| **特色** | 流式/非流式统一框架、零样本多语言、语码切换 |
| **开源** | Apache 2.0 |
| **链接** | [GitHub](https://github.com/FunAudioLLM/CosyVoice), [HuggingFace](https://huggingface.co/FunAudioLLM/CosyVoice2-0.5B) |

**架构详解**：

```
CosyVoice 2 流水线：

文本输入
   │
   ▼
┌──────────────────┐
│ Qwen2.5-0.5B LLM │  ← LLM 骨干做文本到语义 Token 映射
│ (条件语言模型)     │
└────────┬─────────┘
         │ 语义 Token 序列
         ▼
┌──────────────────┐
│ Chunk-Aware       │  ← 分块因果流匹配
│ Causal Flow       │     支持流式 (150ms) 和非流式生成
│ Matching          │
└────────┬─────────┘
         │ Mel 频谱
         ▼
┌──────────────────┐
│ HiFi-GAN 声码器   │  ← 频谱到波形
└────────┬─────────┘
         │
         ▼
      音频输出
```

### 3.2 F5-TTS

> 全非自回归 Flow Matching TTS，速度与质量兼得。

| 项目 | 详情 |
|------|------|
| **团队** | SWivid (开源研究) |
| **论文** | [arXiv:2410.06885](https://arxiv.org/abs/2410.06885) (2024.10) |
| **架构** | 全非自回归；Flow Matching + Diffusion Transformer (DiT) |
| **训练数据** | 100K 小时多语言公开数据 |
| **速度** | RTF 0.15；Fast F5-TTS 仅需 7 步生成，RTF 0.030 |
| **特色** | 零样本语音克隆、无缝语码切换、速度控制 |
| **开源** | 完全开源 |
| **链接** | [GitHub](https://github.com/SWivid/F5-TTS), [项目页](https://swivid.github.io/F5-TTS/) |

**为什么 F5-TTS 重要**：
- 证明了 Flow Matching 在 TTS 中的威力：不需要自回归逐帧生成
- DiT 架构统一了 TTS 和图像/视频生成的技术栈
- 极快的推理速度 (7 步生成) 使边缘部署成为可能

## 四、TTS 技术深度：核心范式详解

### 4.1 VALL-E 范式：神经编解码器语言模型

```
VALL-E (Microsoft, 2023.01) 开创的范式：

核心思路：
  将 TTS 建模为条件语言建模任务
  音频 → 音频编解码器 (如 EnCodec) → 离散 Token → 用 LM 生成

流水线：
  1. 音频编解码器将波形编码为多层离散 Token
  2. 自回归 Transformer 生成第一层 Token (粗)
  3. 非自回归模型生成后续层 Token (细)
  4. 解码器将 Token 还原为波形

突破：
  - 3 秒参考音频即可克隆任意说话人
  - VALL-E 2 达到人类级 TTS (2024.06)
  - 关键技术：Repetition Aware Sampling + Grouped Code Modeling

论文：
  - VALL-E: arXiv:2301.02111
  - VALL-E 2: arXiv:2406.05370
```

### 4.2 Flow Matching 范式

```
Flow Matching 在 TTS 中的应用：

核心思路：
  学习从噪声到目标 Mel 频谱的连续概率流
  比传统扩散模型更直的轨迹 → 更少采样步骤

优势：
  - 推理速度快（7-20 步 vs 扩散模型 50-100 步）
  - 训练稳定
  - 与 DiT 架构天然兼容

代表模型：
  - F5-TTS: 全非自回归 + DiT
  - CosyVoice 2: 分块因果流匹配 (支持流式)
  - Matcha-TTS: OT-CFM (最优传输条件流匹配)
  - SD3 (图像) 和 F5-TTS (语音) 用同一个 Flow Matching 框架
```

### 4.3 LLM 骨干范式

```
用预训练 LLM 做 TTS 骨干：

核心思路：
  复用 LLM 强大的序列建模能力
  LLM 做文本→语义 Token，后接音频解码器

代表：
  - CosyVoice 2: Qwen2.5-0.5B 骨干
  - OpenAI gpt-4o-mini-tts: GPT-4o 骨干 + 自然语言指令控制
  - GPT-4o Voice: 原生多模态模型

优势：
  - 指令可控（用自然语言描述风格）
  - 利用 LLM 的语言理解能力处理复杂文本
  - 与对话系统无缝集成
```
