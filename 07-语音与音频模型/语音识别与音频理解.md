# 语音识别与音频理解

> 语音识别从独立 ASR 模型走向多模态音频理解，GPT-4o Transcribe 大幅降低幻觉率，开源模型 SenseVoice 速度碾压 Whisper。

---

## 一、ASR 技术路线演进

```
ASR 技术路线：

第一代：传统声学模型 (2010s)
  HMM-GMM → DNN-HMM → 端到端 CTC/Attention
  代表：Kaldi, DeepSpeech

第二代：大规模弱监督 (2022-2023)
  用海量弱标签数据训练通用 ASR
  代表：Whisper (100 万小时弱标签数据)

第三代：LLM 原生音频 (2024-2025)  ★★★
  大语言模型直接处理音频输入
  代表：GPT-4o Transcribe, Gemini, Qwen2-Audio
  优势：更强的上下文理解、更少幻觉、音频理解能力

第四代：超快速非自回归 (2024-2025)
  非自回归并行解码，极致速度
  代表：SenseVoice, Paraformer
```

## 二、主要 ASR 模型

### 2.1 OpenAI Whisper v3 / Turbo

> 仍是 2025-2026 最广泛使用的 ASR 模型。

| 项目 | 详情 |
|------|------|
| **公司** | OpenAI (美国) |
| **模型** | Whisper Large-v3 (1.55B), Whisper Large-v3 Turbo (809M, 2024.10) |
| **训练数据** | 100 万小时弱标签音频 + 400 万小时伪标签音频 |
| **语言** | 99+ 语言 |
| **效果** | 干净语音 WER 2.7% |
| **Turbo 版本** | 解码器从 32 层削减到 4 层；比 Large-v3 快 6 倍 (216x 实时)；精度仅降 1-2% |
| **使用量** | HuggingFace 月下载量 410 万次 (2025.12) |
| **注意** | 截至 2026 年初，Whisper v4 尚未发布 |
| **链接** | [GitHub](https://github.com/openai/whisper), [HuggingFace](https://huggingface.co/openai/whisper-large-v3) |

**Whisper 架构回顾**：

```
Whisper 架构：编码器-解码器 Transformer

音频输入 (16kHz)
   │
   ▼ 80-channel Mel 频谱 (30 秒窗口)
┌──────────────────┐
│ 音频编码器        │  ← 2 层 Conv + N 层 Transformer
│ (Encoder)         │     Large-v3: N=32
└────────┬─────────┘
         │ 音频表示
         ▼
┌──────────────────┐
│ 文本解码器        │  ← 自回归 Transformer
│ (Decoder)         │     Large-v3: 32 层, Turbo: 4 层
└────────┬─────────┘
         │
         ▼
   文本 + 时间戳
```

### 2.2 OpenAI GPT-4o Transcribe

> Whisper 的继任者，基于 GPT-4o，大幅减少幻觉。

| 项目 | 详情 |
|------|------|
| **发布** | 2025 年 3 月 |
| **模型** | gpt-4o-transcribe, gpt-4o-mini-transcribe, gpt-4o-mini-transcribe-2025-12-15 |
| **核心改进** | 比 Whisper v2 幻觉率降低约 90%；比早期 GPT-4o transcribe 幻觉率降低约 70% |
| **训练方法** | 强化学习 + 大规模多样化音频数据中期训练 (midtraining) |
| **优势** | 更好处理口音、噪声、变速语音 |
| **输出** | JSON 或纯文本格式 |
| **链接** | [文档](https://platform.openai.com/docs/models/gpt-4o-transcribe), [公告](https://openai.com/index/introducing-our-next-generation-audio-models/) |

**GPT-4o Transcribe vs Whisper**：

| 维度 | Whisper Large-v3 | GPT-4o Transcribe |
|------|-----------------|-------------------|
| 架构 | 独立编码器-解码器 | 基于 GPT-4o 多模态 LLM |
| 幻觉率 | 较高（静音段容易幻觉） | 降低 70-90% |
| 上下文理解 | 有限 | 利用 LLM 上下文能力 |
| 成本 | 开源免费 | API 付费 |
| 部署 | 可本地部署 | 仅 API |

### 2.3 SenseVoice (阿里 FunAudioLLM)

> 比 Whisper 快 7-17 倍的多功能语音理解模型。

| 项目 | 详情 |
|------|------|
| **团队** | 阿里巴巴 FunAudioLLM |
| **论文** | [arXiv:2407.04051](https://arxiv.org/abs/2407.04051) (2024.07) |
| **训练数据** | 40 万+ 小时 |
| **语言** | 50+ 语言 |
| **架构** | 非自回归端到端 |
| **速度** | Small 版：比 Whisper-Small 快 7 倍，比 Whisper-Large 快 17 倍 |
| **多功能** | ASR + 语言识别 (LID) + 语音情感识别 (SER) + 音频事件检测 (AED: 背景音乐、掌声、笑声、哭泣、咳嗽、喷嚏) |
| **效果** | 多语言识别精度超越 Whisper |
| **链接** | [GitHub](https://github.com/FunAudioLLM/SenseVoice), [HuggingFace](https://huggingface.co/FunAudioLLM/SenseVoiceSmall) |

**SenseVoice 多功能输出示例**：

```
输入：一段音频

SenseVoice 输出：
{
  "text": "大家好，今天天气真不错！[笑声]",
  "language": "zh",
  "emotion": "happy",
  "events": ["speech", "laughter", "background_music"]
}

一个模型同时完成 ASR + 情感 + 事件检测
```

### 2.4 Qwen2-Audio (阿里 Qwen)

> 多模态音频理解模型，超越 Gemini 1.5 Pro。

| 项目 | 详情 |
|------|------|
| **团队** | 阿里 Qwen 团队 |
| **论文** | [arXiv:2407.10759](https://arxiv.org/abs/2407.10759) (2024.07) |
| **模型** | Qwen2-Audio-7B, Qwen2-Audio-7B-Instruct |
| **语言** | 8+ 语言/方言 (中文、英文、粤语、法语、意大利语、西班牙语、德语、日语) |
| **两种模式** | Voice Chat (无文字的自由语音交互) + Audio Analysis (音频 + 文字指令) |
| **效果** | AIR-Bench 音频指令遵循评测超越 Gemini-1.5-Pro |
| **架构** | 音频-语言多模态 LLM |
| **链接** | [GitHub](https://github.com/QwenLM/Qwen2-Audio), [HuggingFace](https://huggingface.co/Qwen/Qwen2-Audio-7B-Instruct) |

**Qwen2-Audio vs 传统 ASR**：

```
传统 ASR：音频 → 文字（只做转录）

Qwen2-Audio：音频 + 文字指令 → 任意输出
  例 1: "这段录音里说话人的情绪如何？" → "说话人听起来焦虑不安..."
  例 2: "总结这段会议录音的要点" → "会议讨论了三个议题..."
  例 3: "这是什么乐器在演奏？" → "这是一段大提琴独奏..."

从"转录工具"变为"音频理解助手"
```

## 四、关键技术问题

### 4.1 ASR 幻觉问题

```
Whisper 的幻觉问题：

现象：
  - 在静音或噪声段生成不存在的文字
  - 重复输出相同句子
  - 在音频末尾生成额外内容

原因：
  - 自回归解码器在缺少明确音频信号时倾向于"编造"
  - 弱监督训练数据中存在对齐噪声

GPT-4o Transcribe 的解决方案：
  - 强化学习训练减少幻觉
  - 更好的音频-文本对齐
  - 利用 LLM 上下文理解能力区分有效语音和噪声
  - 幻觉率降低 70-90%
```

### 4.2 实时流式 vs 离线转录

| 维度 | 流式 (Streaming) | 离线 (Offline) |
|------|-----------------|---------------|
| 延迟 | 100-500ms | 秒到分钟 |
| 精度 | 略低 (缺少未来上下文) | 更高 |
| 适用 | 实时字幕、语音助手 | 会议记录、字幕制作 |
| 代表 | Paraformer, CosyVoice 2 | Whisper, GPT-4o Transcribe |
