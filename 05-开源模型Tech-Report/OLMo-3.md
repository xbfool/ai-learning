## 预训练方法 ★★★

### 三阶段预训练

```
阶段 1: 主训练 (Main Training)
  ├── 数据：Dolma 3 通用混合 (~5.9T tokens)
  ├── 重点：构建广泛的语言理解能力
  └── 规模：大部分计算花在这里

阶段 2: 中训练 (Mid-Training) ★
  ├── 数据：加码数学、代码、阅读理解
  ├── 重点：提升推理相关能力

阶段 3: 长上下文扩展 (Long-Context Extension)
  ├── 数据：超长文档
  ├── 重点：扩展上下文窗口
  └── 方法：渐进式扩展 (类似 Llama 3.1)
```

```
Common Crawl (~9.3T tokens)
  ↓ 过滤和清洗
最终混合 (~5.9T tokens)
  ├── 网页文本 (大部分)
  ├── 代码 (重点加码)
  └── 数学/科学 (重点加码)
```

- 6T tokens 用于 32B 模型的预训练

## 深入解读：三阶段预训练详解 ★★★

OLMo 3 的三阶段预训练（Pretraining → Midtraining → Long-Context Extension）是其核心训练策略，每个阶段使用不同的数据集、不同的学习率调度、不同的训练目标。下面逐阶段展开。

### 阶段 1：主训练 (Pretraining on Dolma 3 Mix)

**数据规模与组成**：
- 从 Dolma 3 的 ~9.3T token 语料池中，精选构建了 **5.9T token** 的 Dolma 3 Mix 预训练混合数据集
- 32B 模型在此阶段使用约 **6T tokens**
- 数据来源：
  - **Common Crawl 网页文本**（占比最大）：经过多层过滤和去重后的高质量网页
  - **olmOCR 学术 PDF**：约 **9720 亿 tokens**，来自 1.01 亿篇学术论文，使用 AI2 自研的 olmOCR 流水线将 PDF 转为结构化文本
  - **代码数据**：GitHub 代码仓库，比 Dolma 2 显著加码
  - **数学/科学数据**：数学问题与解答、科学文献，比例大幅提高
  - **百科全书文本**：维基百科等高质量百科内容

**训练策略**：
- 目标是构建广泛的语言理解和知识基础
- 绝大部分计算资源花在这个阶段
- 采用 Warmup-Stable-Decay (WSD) 学习率调度策略：先线性预热 → 稳定阶段维持峰值学习率 → 最后衰减至零
- 使用文档打包（document packing）和文档内掩码（intra-document masking）来避免跨文档注意力干扰

**关键创新**：
- 与 Dolma 2 相比，**大幅增加了代码和数学数据的比例**，这是 OLMo 3 在推理能力上超越前代的关键因素之一
- 质量感知上采样（Quality-aware upsampling）：使用截断幂指数曲线来选择性地重新引入高质量内容，让模型对高质量数据看更多次

### 阶段 2：中训练 (Midtraining on Dolma 3 Dolmino)

中训练是 OLMo 3 最具特色的创新之一，是在主训练和后训练之间插入的一个**能力定向增强**阶段。

**数据规模与组成**：
- 从 **~2.2T token** 的高质量数据池中采样 **100B tokens**（32B 模型使用 100B）
- 数据混合比例（Dolmino Mix）：
  - **28% 网页文本**：经过质量评分筛选的高质量网页
  - **20% 代码**：高质量编程相关数据
  - **19% 数学**：数学问题、证明、解答
  - **14% 问答 (QA)**：阅读理解和问答数据
  - **8% 推理 (Thinking)**：包含推理链（reasoning traces）的数据
  - **6% 指令 (Instruction)**：指令遵循数据
  - **5% PDF 文档**：学术和技术文档

**训练策略**：
- 并行训练：中训练在 **512 个 GPU 上运行两个并行的训练任务**，每个覆盖 100B tokens
- 训练时间极短：总计约 **1.5 天**
- 训练完成后，进行**模型合并**（model merging）并评估，选择最佳检查点
- 学习率在中训练开始时**重新预热**，然后按照新的调度衰减

**核心发现**：
- 增加数学和代码数据的比例可以显著提升推理能力，但**以其他领域性能下降为代价**
- 因此团队精心调整了数据混合比例，在推理增强和通用能力保持之间取得平衡
- 推理链数据（thinking traces）的加入，为后续 Think 模型的训练奠定了基础
- 中训练的 100B tokens 相对于主训练的 6T tokens 只是 **~1.7%**，但对最终性能的提升极为显著

### 阶段 3：长上下文扩展 (Long-Context Extension on Dolma 3 Longmino)

**数据规模与组成**：
- 从 **600B token** 的 Dolma 3 Longmino 数据池中采样
- 7B 模型使用 **50B tokens**，32B 模型使用 **100B tokens**
- 数据混合比例：**34% 长上下文数据 + 66% 短上下文数据**（即 1:2 比例）
- 长上下文数据来源：
  - 合成长文档数据
  - olmOCR 处理的学术 PDF 语料（天然的长文档）
- 短上下文数据：直接复用中训练阶段的数据

**上下文扩展方法**：
- 将上下文窗口从 **8K tokens 扩展到 ~65K tokens**
- 位置编码扩展使用 **YaRN 方法**（Yet another RoPE extensioN）
- 团队测试了多种 RoPE 扩展方案：调整基频缩放、位置插值、YaRN 等
- 最终发现：**只在全注意力层（Full Attention layers）应用 YaRN 效果最佳**
- 训练时采用文档打包（document packing）和文档内掩码（intra-document masking），确保模型学习的是单文档内的长距离依赖，而非跨文档的虚假关联

**为什么混合长短数据**：
- 纯长文档训练会导致短文档任务性能退化
- 1:2 的混合比例在长上下文能力和短上下文保持之间取得了最佳平衡

## 深入解读：Dolma 3 数据流水线 ★★★

Dolma 3 是目前最大且最透明的开源预训练数据集之一，其数据处理流水线的每个环节都值得细致分析。

### 数据来源

```
Dolma 3 语料池 (~9.3T tokens)
├── Common Crawl 网页 (主体)
│   ├── 多个 CC 快照
│   └── 尊重 robots.txt 规则，排除付费墙内容
├── olmOCR 学术 PDF (~972B tokens)
│   ├── 1.01 亿篇学术论文
│   └── AI2 自研 OCR 流水线处理
├── GitHub 代码仓库
│   └── 代码数据（多语言、多仓库）
├── 数学数据
│   └── 数学问题、解答、证明
└── 百科全书文本
    └── 维基百科等参考资料
```

### 过滤策略

Dolma 3 的过滤是多层次、多维度的：

**1. URL 级过滤**：
- 基于 URL 黑名单排除已知低质量站点
- 排除明确禁止爬取的站点（遵循 robots.txt 规则）
- 排除付费墙内容和受版权保护的来源

**2. 文档级过滤**：
- **语言检测**：使用语言识别工具过滤非目标语言文档
- **质量评分**：使用训练好的分类器对文档质量进行打分
- **安全过滤**：基于内容安全分类器过滤有害内容
- **长度过滤**：排除过短或过长的异常文档

**3. 段落级过滤**：
- 在文档级过滤之后，还有**段落级别的精细过滤**
- 移除重复段落、低质量段落、广告内容等

### 去重方法

Dolma 3 使用了名为 **Duplodocus** 的去重框架，在万亿 token 规模上保证数据唯一性：

```
去重流水线：
1. URL 去重 → 完全相同 URL 的文档只保留一份
   ↓
2. 文档级去重 → 基于内容哈希的近似去重
   ↓
3. 质量过滤和内容过滤
   ↓
4. 段落级去重 → 最后一步，使用 Bloom Filter 的段落去重
```

**技术实现**：
- 使用 **Rust 实现的 Bloom Filter** 进行高效去重（这是 Dolma Toolkit 的核心能力）
- URL 去重和文档级去重**先于**质量过滤执行，段落级去重**最后**执行
- 全局去重：不是在单个 CC 快照内去重，而是跨所有快照进行全局去重
- 基准测试去污染（decontamination）：确保训练数据中不包含评估基准的内容

### 代码和数学数据的特殊处理

**代码数据**：
- 来源于 GitHub 公开代码仓库
- 经过质量评分和去重处理
- 在 Dolma 3 中代码数据比例相比 Dolma 2 **大幅提升**
- 这是 OLMo 3 在代码任务上显著进步的直接原因

**数学数据**：
- 包含数学问题、解答过程、数学证明
- 包含推理链（reasoning traces）格式的数据
- 在中训练数据（Dolmino）中，数学数据占比高达 **19%**

**olmOCR 学术 PDF 数据**：
- AI2 开发了专门的 **olmOCR 工具**来将学术 PDF 转换为高质量文本
- 处理了 1.01 亿篇学术论文，生成约 972B tokens
- 这是 Dolma 3 的重大创新——很少有开源数据集能大规模处理 PDF 并公开
- olmOCR 工具本身也是开源的（github.com/allenai/olmocr）

### 质量感知上采样

Dolma 3 引入了一种**质量感知上采样**技术：

- 使用截断幂指数曲线（truncated power-exponential curves）对数据进行质量评分
- 高质量数据被上采样（即在训练中出现更多次）
- 低质量数据被下采样或完全排除
- 这使得模型在有限的训练预算下能看到更多高质量内容

### Dolma Toolkit

所有数据处理都由开源的 **Dolma Toolkit** 完成，支持：
- 并行数据处理（适应万亿 token 规模）
- 语言过滤和质量过滤
- 安全内容过滤
- 基于 Rust 的 Bloom Filter 快速去重
- 预构建的标注器（taggers）套件
- 完全可审计和可复现

## 深入解读：OLMo 3-Think 推理模型训练 ★★★

OLMo 3-Think 是目前最强的**完全开源推理模型**，其训练流水线采用了三阶段后训练方法：SFT → DPO → RLVR。

### 训练流水线总览

```
OLMo 3 Base (预训练 + 中训练 + 长上下文)
    ↓
阶段 A: 思维 SFT (Thinking SFT)
    ├── 数据：Dolci-Think SFT 数据集
    ├── 内容：数学、代码、通用推理、安全任务
    └── 目标：教会模型外化推理过程（生成 thinking tokens）
    ↓
阶段 B: 思维 DPO (Thinking DPO)
    ├── 数据：选择/拒绝回答对
    ├── 来源：不同规模 Qwen 模型生成的回答对
    └── 目标：从好坏推理链的差异中学习
    ↓
阶段 C: RLVR (Reinforcement Learning with Verifiable Rewards)
    ├── 奖励：符号化数学求解器 + 单元测试
    ├── 方法：正确答案获得正奖励，错误答案获得 0
    └── 目标：在困难数学任务上大幅提升准确率
```

### 阶段 A：思维 SFT（Supervised Fine-Tuning）

**Dolci-Think SFT 数据集**：
- Dolci 是 AI2 为 OLMo 3 专门构建的后训练数据套件
- Dolci-Think 是其中面向推理能力的子集
- 包含的任务类型：
  - **数学推理**：多步数学问题及详细解答过程
  - **代码推理**：编程问题及思考链
  - **通用问题解决**：需要逻辑推理的通用任务
  - **安全关键任务**：涉及安全判断的推理场景
- 目标：教会模型**外化推理过程**，即将内部"思考"以显式的文本形式表达出来

### 阶段 B：思维 DPO（Direct Preference Optimization）

**数据构建**：
- 使用不同规模的 Qwen 模型生成同一问题的回答
- 更大/更好的模型生成 "chosen"（优选）回答
- 更小/更差的模型生成 "rejected"（拒绝）回答
- 模型从好坏推理链的**差异（delta）**中学习

**训练目标**：
- 提升推理链的质量，而不仅仅是最终答案的正确性
- 让模型学会更清晰、更有逻辑的推理路径
- DPO 相比传统 RLHF 更稳定、更高效

### 阶段 C：RLVR（Reinforcement Learning with Verifiable Rewards）

这是 Think 模型训练中最关键的阶段，也是区别于普通指令微调模型的核心。

**核心思想**：
- 传统 RLHF 依赖奖励模型（Reward Model），而 RLVR **不需要奖励模型**
- RLVR 使用**可验证的奖励信号**：
  - **数学题**：使用符号化数学求解器（symbolic math solvers）验证答案是否正确
  - **代码题**：使用单元测试（unit tests）验证代码是否正确执行
- 奖励函数极其简单：正确 → 正奖励，错误 → 0 奖励

**训练规模**：
- OLMo 3 32B Think 在初始训练后，团队又**继续了额外 21 天的 RL 训练**
- 使用 **224 个 GPU**
- 在 Dolci-Think-RL 数据集上进行多轮迭代
- 这种长时间的 RL 训练显著提升了困难数学任务（如 AIME）上的表现

**RLVR 的优势**：
- **完全可复现**：不依赖主观的人类偏好标注
- **信号清晰**：对错二值反馈，无噪声
- **可扩展**：只要有可验证的任务，就可以无限扩展训练数据
- **开源友好**：整个训练过程不需要闭源的奖励模型

### 性能表现

| 基准 | OLMo 3-Think 7B | OLMo 3-Think 32B |
|------|-----------------|------------------|
| MATH | 接近 Qwen 3 8B | 同尺寸最强完全开源 |
| AIME 2024/2025 | 与 Qwen 3 8B 差距数个百分点 | 极具竞争力 |
| HumanEvalPlus | 领先所有同类比较模型 | 代码推理最强 |
| 通用推理 | 完全开源中最强 | 仅略低于 Qwen 3 32B Thinking |

## 深入解读："完全开源" vs "仅开源权重" ★★★

OLMo 3 代表的"完全开源"与 Llama、Qwen 等代表的"仅开源权重"有着本质区别，这种区别对研究者和行业的实际价值不可忽视。

### 开放程度对比

```
                    完全开源            仅开源权重         闭源
                   (OLMo 3)        (Llama/Qwen)     (GPT-4/Claude)
─────────────────────────────────────────────────────────────
模型权重              ✅                ✅                ❌
推理代码              ✅                ✅                ❌
训练代码              ✅                ❌                ❌
预训练数据            ✅                ❌                ❌
训练检查点（全过程）   ✅                ❌                ❌
数据处理代码          ✅                ❌                ❌
评估代码              ✅                部分               ❌
训练超参数            ✅                部分               ❌
─────────────────────────────────────────────────────────────
可复现训练            ✅                ❌                ❌
可审计数据            ✅                ❌                ❌
可研究训练动态        ✅                ❌                ❌
```

### 对研究者的实际价值

**1. 训练动态研究**：
- OLMo 3 公开了训练过程中所有中间检查点
- 研究者可以观察模型在训练过程中的行为变化
- 例如：某种能力是在什么时候涌现的？训练数据的变化如何影响模型行为？
- 这种研究用仅开源权重的模型**完全无法实现**

**2. 数据影响研究**：
- 完整公开 Dolma 3 数据集意味着研究者可以：
  - 研究特定数据对模型能力的影响
  - 分析数据偏见（bias）的来源
  - 设计更好的数据过滤和混合策略
- 对于 Llama 和 Qwen，由于训练数据不公开，这类研究只能猜测

**3. 可复现性**：
- 学术研究的核心要求之一就是可复现性
- OLMo 3 是目前**唯一**可以从零完全复现的前沿级别 LLM
- 包括从原始数据处理开始的全流程复现

**4. 消融实验**：
- 有了完整的训练代码和数据，研究者可以进行消融实验
- 例如：移除某类数据后模型性能如何变化？换一种学习率调度会怎样？

### 对行业的实际价值

**1. 合规审计**：
- 在监管日益严格的环境下（如欧盟 AI Act），公司需要证明其 AI 系统的训练数据是合规的
- OLMo 3 的完全透明性使得合规审计成为可能
- 使用 Llama 或 Qwen 的公司**无法证明**训练数据中不包含敏感内容

**2. 领域定制**：
- 有了完整的训练代码和数据处理流水线，企业可以：
  - 用自己的数据从头训练
  - 修改数据混合比例以适应特定领域
  - 在完全了解底层数据的情况下做微调

**3. 信任建设**：
- 在高风险应用场景（医疗、金融、法律）中，完全的训练透明性是建立信任的基础

### 性能代价与权衡

值得注意的是，完全开源也有其代价：

- **数据限制**：由于需要公开所有训练数据，某些受限数据源（如某些专有数据集）必须被排除，这可能部分解释了 OLMo 3 与 Qwen 3 之间的性能差距
- **资源投入**：构建和维护完全开放的数据流水线需要巨大的工程投入
- OLMo 3 团队的立场是：**透明性的价值远大于由此带来的微小性能损失**

### 训练方法论对比

**预训练数据规模**：
- **OLMo 3 32B**：~6T tokens（Dolma 3 Mix 5.9T + 中训练 100B + 长上下文 100B）
- **Llama 3.1**：约 15T+ tokens
- **Qwen 2.5**：约 18T tokens
- OLMo 3 用了**明显更少的数据**，但通过三阶段训练策略和高质量数据弥补了部分差距

**训练策略**：
- **OLMo 3**：三阶段策略（主训练 → 中训练 → 长上下文），中训练是核心创新
- **Llama 3.1**：大规模预训练 + 长上下文扩展 + RLHF
- **Qwen 2.5**：大规模预训练 + 多阶段后训练

**推理能力**：
- **OLMo 3-Think**：SFT → DPO → RLVR，完全开源的推理训练
- **Llama 3.1**：无专门推理版本（Llama 3.3 时才有推理增强）
- **Qwen 2.5**：Qwen 2.5 本身无 Think 版本，但 Qwen 3 系列有 Thinking 模型

### 性能对比

**基座模型**（32B 规模）：
- OLMo 3 Base 在多数基准上**达到或超越** Qwen 2.5 和 Llama 3.1（同尺寸）
- 在完全开源模型中**性能最强**
- 考虑到训练数据量仅为对手的 1/3，这一成绩更加令人印象深刻

**推理模型**：
- OLMo 3-Think 32B 是同尺寸中**最强的完全开源推理模型**
- 在数学（MATH）和代码（HumanEvalPlus）上极具竞争力
- 与 Qwen 3 32B Thinking 相比，在 MMLU 等通用知识基准上仍有差距
- 但在 GPQA 等科学推理任务上表现不俗

**核心启示**：
- OLMo 3 证明了"完全开源"的模型在性能上可以与"仅开源权重"的模型竞争
- 虽然用了更少的训练数据（6T vs 15-18T），但通过精心的数据策略和中训练，几乎弥合了差距
- 这为整个开源 AI 社区设立了新的标准：真正的开源应该包含数据和训练流程，而不仅仅是权重
