# GLM-5 技术报告解读

> 智谱 AI 的新一代旗舰模型，在 10 万张华为昇腾 910B 上训练，零 NVIDIA 芯片。

## 基本信息

- **开发者**：智谱 AI (Zhipu AI)
- **发布时间**：2026 年 2 月
- **官网**：[glm5.net](https://glm5.net/)
- **协议**：MIT License
- **技术报告知乎解读**：[知乎](https://zhuanlan.zhihu.com/p/2009393352567697921)

## 架构

| 参数 | 值 |
|------|-----|
| 总参数 | ~744B (745B) |
| 激活参数 | ~44B |
| 架构 | MoE (Mixture of Experts) |
| 预训练数据 | 28.5T tokens |
| 上下文窗口 | 200K (输入) |
| 最大输出 | 131K tokens |
| 注意力机制 | 采用 DeepSeek 稀疏注意力 |

## 国产化训练 ★★★

### 硬件
- **100,000 张华为昇腾 (Ascend) 910B**
- 零 NVIDIA 芯片
- 零 AMD 芯片
- 这是中国 AI 自主化的重要里程碑

### 意义
- 证明国产芯片可以支撑前沿大模型训练
- 昇腾 910B 在大规模分布式训练中的可用性
- 为更多国产芯片训练提供了路径参考

## 核心能力

### 编程 ★★
- **SWE-bench Verified：77.8%** — 开源模型最高
  - 任务：修复真实 GitHub 仓库中跨数千行代码的 Bug
  - 需要理解大型代码库 + 定位问题 + 生成补丁
- 编程任务性能比前代提升 **20%+**
- 用户体验接近 Claude Opus 4.5

### Agent 能力
- 专注于 Agentic Engineering
- 强大的工具调用和多步骤执行
- 适合构建复杂的自动化工作流

## 技术特点

### 稀疏注意力
- 采用 DeepSeek 的稀疏注意力机制
- 有效降低长上下文的计算开销
- 支持 200K 输入上下文

### 大规模预训练数据
- 28.5T tokens，是目前公开的最大预训练数据量之一
- 对比：DeepSeek-V3 (14.8T), Kimi K2 (15.5T), Llama 3 (15T)

## 性能定位

- 与 GPT-5.2 级别竞争
- 开源模型中 SWE-bench 第一
- 编程/Agent 是核心优势方向

## 核心贡献总结

1. **国产芯片训练**：10 万张昇腾 910B，零 NVIDIA
2. **最大预训练数据**：28.5T tokens
3. **SWE-bench 开源 SOTA**：77.8%
4. **编程+Agent 导向**：定位清晰
5. **MIT 开源**：最宽松的开源协议

## 值得深入研究的部分
- 昇腾 910B 上的分布式训练策略
- 如何解决国产芯片的软件生态问题
- 稀疏注意力的具体实现
- 28.5T token 的数据处理流水线
