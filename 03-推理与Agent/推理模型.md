# 推理模型

> 推理模型通过 RL 训练获得 "思考" 能力，是 2024-2026 最重要的突破。

## 一、核心概念

### 什么是推理模型？
- 在生成最终答案之前，先进行长时间的 **内部思考** (Chain-of-Thought)
- 思考过程可能包含：
  - 问题分解
  - 多种方案探索
  - 自我验证和纠错
  - 反思和回溯
- 思考时间与问题难度成正比

### Test-Time Compute Scaling
- **传统 Scaling**：模型越大 → 性能越好 (Kaplan/Chinchilla)
- **新 Scaling**：思考越久 → 性能越好 (Test-Time Compute)
- **关键论文**：[Scaling LLM Test-Time Compute](https://arxiv.org/abs/2408.03314)

- 开源，基于 DeepSeek-V3 (671B MoE)
- **R1-Zero**：纯 RL 训练（GRPO），无 SFT
  - 发现：RL 自动涌现 CoT、自我验证、反思
  - "Aha moment"：训练过程中模型突然 "领悟" 新的推理策略
- **R1**：冷启动 SFT + GRPO + 拒绝采样 + 最终 SFT
  - 流程更完整，效果更好
- **蒸馏**：R1 的能力成功蒸馏到 Qwen/Llama 的 1.5B-70B 模型
- **训练流水线**：
  ```
  DeepSeek-V3 Base
    → 冷启动 SFT (少量高质量 CoT 数据)
    → GRPO RL 训练 (数学/代码用规则奖励)
    → 拒绝采样 (用训练好的模型生成高质量数据)
    → 最终 SFT (在拒绝采样数据上微调)
    → DeepSeek-R1
  ```

## 三、推理模型训练方法论

### 方法 1：GRPO (Group Relative Policy Optimization)
```
适用场景：数学、代码等有客观答案的领域

训练循环：
  for each prompt:
    1. 采样 G 个回答 (带完整 CoT)
    2. 验证最终答案（数学用规则，代码用执行）
    3. 计算组内相对优势
    4. 优化策略（鼓励正确推理路径）

奖励设计：
  - 答案正确 → r = 1
  - 答案错误 → r = 0
  - (可选) 格式正确 → 额外 r
```

### 方法 2：过程奖励模型 (PRM)
```
适用场景：需要监督推理过程的场景

核心思想：
  不只奖励最终结果，还奖励每个推理步骤
  → 更快的信用分配（哪一步推理错了？）
  → 更高的样本效率

论文：
  "Let's Verify Step by Step" (OpenAI, 2023)

挑战：
  - 需要步骤级别的标注数据（成本高）
  - 或者自动化的步骤验证器
```

### 方法 3：蒸馏 (Distillation)
```
从大推理模型蒸馏到小模型：
  1. 大模型生成高质量 CoT 推理轨迹
  2. 用这些轨迹对小模型做 SFT
  3. (可选) 在小模型上继续 RL 训练

DeepSeek-R1 蒸馏结果：
  - Qwen-32B + R1 蒸馏 → 超越 o1-mini
  - Qwen-7B + R1 蒸馏 → 超越 QwQ-32B-Preview
  - 效果惊人，证明推理能力高度可迁移
```

## 四、涌现现象与洞察

### RL 训练中的涌现行为
1. **自我验证**："Let me check this result..."
2. **反思**："Wait, I made an error. Let me reconsider..."
3. **多路径探索**："Alternatively, I could approach this by..."
4. **元认知**："This problem seems similar to..., I should try..."

### 关键洞察
- RL 能发现 SFT 无法教授的推理策略
- 推理能力不仅仅是 "格式"（CoT 模板），更是 "行为模式"
- 可验证奖励（数学正确性、代码通过）是训练推理的最佳奖励信号
- 推理能力具有很好的**跨领域迁移性**
